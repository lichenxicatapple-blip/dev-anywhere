# 0.5 Readiness

`0.5.0` 是 DEV Anywhere 第一个可对外公开的稳定版本。它不是某个单点主题版本；只有核心平台、核心链路和发布门禁都稳定后，才切 `0.5.0`。

## 发布定位

- `0.4.x`：内部硬化线。允许继续频繁发 patch，用来修复 iPad、移动端 PTY、会话生命周期、上传下载预览和发布链路问题。
- `0.5.0`：第一个公开稳定版。不得夹带未经充分验证的大功能。
- `0.5.x`：稳定线上的小修复。原则上不引入协议破坏和大面积交互重构。

## 准入原则

- 不靠主观感觉判断“差不多能发”；每条关键链路都要有自动化或明确人工验收记录。
- iPad 不是“放大版手机”，也不是“小号桌面”。iPad + 外接键盘是独立准入场景。
- 新功能让位于稳定性。除非直接服务于稳定性、可观测性或发布门禁，否则推迟到 `0.5.0` 之后。
- 发现 flaky 就按失败处理。允许诊断性 skip，但默认发布门禁不能依赖“偶尔过”。

## 体验准入矩阵

| 场景            | 必过链路                                                                                    |
| --------------- | ------------------------------------------------------------------------------------------- |
| 桌面浏览器      | 选择开发机、创建 Agent 会话、创建终端会话、恢复会话、审批、上传、下载、图片预览、客户端管理 |
| 手机浏览器      | 会话列表、创建会话 sheet、PTY 输入、PTY 长按复制、回到底部、设置弹窗、文件选择、图片预览    |
| iPad 竖屏       | 侧边栏/会话列表、创建会话、PTY 输入与选择、设置子弹窗、图片预览                             |
| iPad 横屏       | brand 区域不挤压内容、侧边栏底部按钮完整、PTY 可读可操作、弹窗有安全边距                    |
| iPad + 外接键盘 | 不误弹软键盘、焦点可预测、常用快捷键不破坏 PTY 输入、触摸选择仍可用                         |
| 线上部署        | VPS Web/Relay 版本一致，本机全局包版本一致，default/cloud serve 可连接 relay                |

## 核心链路门禁

### 会话生命周期

- 创建 Agent 会话成功，失败时错误文案明确且不丢当前表单。
- 创建纯终端会话成功，终止终端会话的文案不提 agent。
- 恢复历史会话、切换会话、刷新页面后状态一致。
- proxy serve 重启不应打断 web-created 纯终端会话。

### PTY 体验

- PTY 渲染遵循远端输出，不做额外浅色/深色重映射。
- 回到底部按钮不遮挡底部操作区，在手机和 iPad 上位置稳定。
- 手机和 iPad 长按复制不误聚焦输入框，不因软键盘变化导致选区漂移。
- 横向滚动、触摸回看、持续输出、审批状态和 reconnect 都不把用户强行拉回底部。

### 文件链路

- 上传走 Relay HTTP PUT 正式链路。
- 下载和图片预览走 Relay HTTP streaming 正式链路。
- 大文件下载链接有效期足够，不在正常下载等待中失效。
- 选中文本兜底下载/预览可用，避免自动链接误识别时无路可走。

### 设置和客户端管理

- 所有设置子弹窗标题、副标题、内容密度、按钮尺寸保持一致。
- 客户端管理展示用户能理解的设备/浏览器/开发机信息，避免 hash 噪音。
- 断开其他客户端、刷新、当前设备等操作的大小、位置、语气一致。

## 验证门禁

`0.5.0` 候选必须跑过：

```bash
pnpm quality:check
pnpm release:check
pnpm release:smoke
```

针对 iPad 和移动 PTY 改动，还必须至少跑过：

```bash
TEST_MOBILE_REQUIRE_EMULATOR=1 pnpm test:mobile:serial apps/web/e2e/mobile/pty-long-press-copy.spec.ts
TEST_MOBILE_REQUIRE_EMULATOR=1 pnpm test:mobile:serial apps/web/e2e/mobile/pty-input-keyboard.spec.ts
pnpm test:layout
```

人工验收至少覆盖：

- iPad Safari 竖屏截图。
- iPad Safari 横屏截图。
- iPad Safari 横屏打开 PTY 会话截图。
- iPad + 外接键盘下 PTY 输入、长按复制、切换会话。
- 手机 Safari/Chrome 下 PTY 输入、长按复制、图片预览。

## 当前出线标准

只有同时满足以下条件，才允许发布 `0.5.0`：

- 没有已知 P0/P1 稳定性问题。
- 没有会阻断 iPad/手机/桌面核心链路的 P2 问题。
- 发布门禁无失败、无 flaky。
- 线上 VPS、npm 包、本机全局包能升级到同一版本。
- `CHANGELOG.md` 明确写出 `0.5.0` 的稳定性边界和仍不承诺的内容。
