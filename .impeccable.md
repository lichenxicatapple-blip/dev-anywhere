## Design Context

### Users

DEV Anywhere is used by developers who need browser access to a local or remote development runtime. On mobile, the main jobs are monitoring active coding sessions, approving or denying tool use, sending short prompts, switching sessions, and using the terminal for emergency operations when a laptop is not available.

### Brand Personality

Calm, technical, dense. The interface should feel like a trustworthy developer console: focused, compact, legible, and operational. It should not feel like a marketing page, a consumer chat app, or a decorative dashboard.

### Aesthetic Direction

Keep the existing dark, VS Code-adjacent developer palette and compact information density. Mobile layouts should be touch-first and intentionally designed instead of scaled-down desktop panels. Use bottom sheets, full-page navigators, staged flows, sticky action regions, and keyboard-aware surfaces where they fit the job better than desktop dialogs or popovers.

### Design Principles

1. Build the mobile product surface deliberately; do not stack local CSS patches until narrow bugs disappear.
2. Preserve desktop power while giving mobile its own interaction model.
3. Keep critical capabilities available on mobile, using progressive disclosure rather than hiding important actions.
4. Use measurable contracts for mobile quality: no document overflow, 44px touch targets, safe-area handling, keyboard-safe inputs, and orientation-safe terminal layout.
5. Treat PTY, approvals, session switching, and short replies as first-class mobile workflows.
6. Prefer replacing fragile component internals over adding conditional class noise to components that were designed for desktop.
