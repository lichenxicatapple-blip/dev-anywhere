   Rewrite from scratch based on feishu version. Remove ALL Taro code (Taro.connectSocket, IS_H5, TaskLike interface, createNativeTask wrapper). Use native WebSocket directly.

   Class `WebSocketManager` with private fields:
   - `ws: WebSocket | null = null`
   - `url = ''`
   - `connected = false`
   - `closed = false`
   - `reconnectTimer: ReturnType<typeof setTimeout> | null = null`
   - `reconnectAttempt = 0`
   - `messageHandlers = new Set<(data: string) => void>()`
   - `binarySubscribers = new Map<string, Set<(data: Uint8Array) => void>>()`
   - `statusHandlers = new Set<(connected: boolean) => void>()`
   - `pendingQueue: string[] = []`

   Public methods (preserve feishu API surface):
   - `connect(url: string): void` - close existing ws/timer, reset state, set `this.url = url`, `this.closed = false`, `this.reconnectAttempt = 0`, call `doConnect()`
   - `send(data: string): boolean` - if no ws return false with console.warn; if not connected, push to pendingQueue and return false; else call doSend and return true
   - `close(): void` - set `this.closed = true`, clear reconnectTimer, close ws, set `this.connected = false`
   - `onMessage(handler: (data: string) => void): () => void` - add to messageHandlers, return unsubscribe
   - `onStatusChange(handler: (connected: boolean) => void): () => void` - add to statusHandlers, return unsubscribe
   - `isConnected(): boolean` - return `this.connected`

   New public method (per D-06):
   - `subscribeBinary(sessionId: string, handler: (data: Uint8Array) => void): () => void` - add handler to `binarySubscribers.get(sessionId)` Set (create Set if absent), return unsubscribe function that removes handler and deletes Set if empty

   Private methods:
   - `doConnect(): void` - create `new WebSocket(this.url)`, set `ws.binaryType = 'arraybuffer'`, register event listeners:
     - `ws.addEventListener('open', ...)`: set connected=true, reconnectAttempt=0, flushPendingQueue(), notify statusHandlers(true)
     - `ws.addEventListener('message', ...)`: if `event.data instanceof ArrayBuffer` call dispatchBinary(new Uint8Array(event.data)), else cast to string and notify messageHandlers
     - `ws.addEventListener('close', ...)`: set connected=false, ws=null, notify statusHandlers(false), if not closed call scheduleReconnect()
     - `ws.addEventListener('error', ...)`: no action (close event follows)
   - `scheduleReconnect(): void` - calculate `delay = Math.min(1000 * Math.pow(2, this.reconnectAttempt), 30000)`, increment reconnectAttempt, set timer to call doConnect()
   - `dispatchBinary(view: Uint8Array): void` - parse binary frame: `if (view.length < 2) return`, `const sidLen = view[0]`, `if (view.length < 1 + sidLen) return`, `const sessionId = new TextDecoder().decode(view.subarray(1, 1 + sidLen))`, `const ptyData = view.subarray(1 + sidLen)`, look up `binarySubscribers.get(sessionId)` and call each handler with `ptyData`
   - `doSend(data: string): void` - `this.ws?.send(data)`
   - `flushPendingQueue(): void` - splice all from pendingQueue, call doSend for each

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | T (Tampering) | websocket.ts binary frame | mitigate | dispatchBinary validates frame structure (length >= 2, sidLen valid) before parsing. Malformed frames silently dropped. |
| T-08-04 | D (Denial of Service) | websocket.ts reconnect | mitigate | Exponential backoff (1s to 30s cap) prevents reconnect storm. close() sets closed flag to stop all reconnection. |
| T-08-05 | S (Spoofing) | relay-client register | accept | clientId is non-secret identifier. No auth system per project scope (personal-use tool in trusted environment). |
| T-08-06 | I (Information Disclosure) | websocket.ts message handlers | accept | JSON messages logged at warn level only on parse failure. No PII in protocol messages. |
