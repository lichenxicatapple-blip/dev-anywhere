    **Part A: Add binary frame passthrough to relay proxy handler (D-07, D-42)**

    In `apps/relay/src/handlers/proxy.ts`:

    The `ws` library's `message` event signature is `(data: Buffer | ArrayBuffer | Buffer[], isBinary: boolean)`. Currently the handler does `const raw = data.toString()` for all messages.

    Add binary frame handling BEFORE the existing JSON parsing:
    ```typescript
    proxyWs.on("message", (data: Buffer, isBinary: boolean) => {
      if (isBinary) {
        // D-07: Binary frame passthrough - only read sessionId for routing
        if (data.length < 2) return; // malformed
        const sessionIdLen = data[0];
        if (data.length < 1 + sessionIdLen) return; // malformed
        const sessionId = data.subarray(1, 1 + sessionIdLen).toString("utf-8");

        // Route to all clients viewing this session
        // D-42: zero-copy, forward entire buffer including sessionId prefix
        const clients = registry.getClientsForSession(sessionId);
        for (const clientWs of clients) {
          if (clientWs.readyState === WebSocket.OPEN) {
            clientWs.send(data);
          }
        }
        return;
      }
      // Existing JSON message handling continues below
      const raw = data.toString();
      // ...
    });
    ```

    Also add binary frame length validation: `sessionIdLen` must be < 256 and `data.length` must be < 10MB (sanity check against DoS).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Tampering | Binary frame sessionId spoofing | mitigate | Relay validates sessionIdLen < 256 and sessionId matches a known session in registry. Unknown sessionIds are dropped silently. |
| T-09-07 | DoS | Oversized binary frame | mitigate | Relay checks `data.length < 10MB` before processing binary frame. Frames exceeding limit are dropped with warning log. |
| T-09-08 | Spoofing | Unauthorized binary frame injection | accept | Current trust model: proxy authenticates via proxyId on registration. Binary frames from authenticated proxy are trusted. Phase 9 does not add new auth. |
| T-09-09 | Information Disclosure | PTY data in transit | accept | WebSocket traffic is plaintext unless TLS termination is configured (Nginx). Same trust level as existing JSON messages. Personal dev tool on trusted network. |
