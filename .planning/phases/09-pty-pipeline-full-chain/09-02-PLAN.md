    In `apps/proxy/src/ipc-protocol.ts`, add binary frame support alongside existing NDJSON:

    **New exports:**
    ```typescript
    // IPC binary frame marker byte (D-27: 0x00, cannot appear as first byte of JSON line)
    export const IPC_BINARY_MARKER = 0x00;

    // Encode a binary PTY data frame for IPC transport
    // Format: [1B 0x00 marker][4B payload_len uint32LE][1B sessionId_len][sessionId UTF-8][PTY data]
    export function encodeBinaryIpcFrame(sessionId: string, data: Buffer): Buffer {
      const sessionIdBuf = Buffer.from(sessionId, "utf-8");
      const payloadLen = 1 + sessionIdBuf.length + data.length;
      const frame = Buffer.alloc(1 + 4 + payloadLen);
      let offset = 0;
      frame[offset] = IPC_BINARY_MARKER; offset += 1;
      frame.writeUInt32LE(payloadLen, offset); offset += 4;
      frame[offset] = sessionIdBuf.length; offset += 1;
      sessionIdBuf.copy(frame, offset); offset += sessionIdBuf.length;
      data.copy(frame, offset);
      return frame;
    }
    ```

    **Modify `createIpcReader`** to handle mixed protocol. The reader currently uses LineBuffer for NDJSON. It needs a new state machine that:

    3. If byte !== 0x00: continue NDJSON line buffering (existing behavior)
    4. Calls a new `onBinaryFrame(sessionId: string, data: Buffer)` callback for binary frames

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| proxy -> relay (WebSocket) | Binary PTY data and JSON control messages cross network boundary |
| relay -> client (WebSocket) | Binary frames forwarded to browser clients |
| client -> relay (WebSocket) | Client sends JSON only in Phase 9 (no binary upload) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | Binary frame sessionId spoofing | mitigate | Relay validates sessionIdLen < 256 and sessionId matches a known session in registry. Unknown sessionIds are dropped silently. |
| T-09-05 | DoS | Oversized binary frame | mitigate | Relay checks `data.length < 10MB` before processing binary frame. Frames exceeding limit are dropped with warning log. |
| T-09-06 | Spoofing | Unauthorized binary frame injection | accept | Current trust model: proxy authenticates via proxyId on registration. Binary frames from authenticated proxy are trusted. Phase 9 does not add new auth. |
| T-09-07 | Information Disclosure | PTY data in transit | accept | WebSocket traffic is plaintext unless TLS termination is configured (Nginx). Same trust level as existing JSON messages. Personal dev tool on trusted network. |
| T-09-08 | DoS | IPC binary frame flood | accept | IPC is local Unix socket between processes on same machine. No network exposure. OS-level process isolation applies. |
</threat_model>
