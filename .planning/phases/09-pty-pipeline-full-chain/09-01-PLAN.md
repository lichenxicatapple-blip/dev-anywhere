    **File header (D-29):** 4B 'CCAE' ASCII + 2B version uint16LE = 6 bytes total.

    **Event structure (D-30):** `[1B type][8B timestamp float64LE][4B payload_len uint32LE][NB payload][4B total_len uint32LE]`. The total_len trailer = 17 + payload.length. This enables reverse scanning per D-48.

    **Event types (D-31):**
    - 0x01 PTY_DATA: raw PTY bytes as payload
    - 0x02 SNAPSHOT: SerializeAddon.serialize() UTF-8 output as payload
    - 0x03 RESIZE: fixed 4B payload = [2B cols uint16LE][2B rows uint16LE]
    - 0x04 METADATA: JSON bytes as payload (D-23: must be first event in file)

    **Class interface:**
    ```typescript
    export class EventStore {
      constructor(eventsPath: string);
      open(metadata: { cols: number; rows: number; sessionId: string; createdAt: string }): void;
      appendPtyData(data: Buffer): void;          // D-02: immediate write, D-22: no fragmentation
      appendSnapshot(serialized: string): void;    // D-05: SNAPSHOT event type 0x02
      appendResize(cols: number, rows: number): void;
      shouldSnapshot(): boolean;                   // D-04: true every SNAPSHOT_INTERVAL events
      close(): Promise<void>;                      // D-03: gzip remaining file at session end
      // Recovery (D-17, D-48):
      static findLatestSnapshot(eventsPath: string): Buffer | null;  // reverse scan
      static readEventsAfterSnapshot(eventsPath: string, snapshotOffset: number): Buffer[];
    }
    ```

    **Write strategy (D-02, D-14):** Open fd once with `fs.openSync(path, 'a')`. Use `fs.writeSync(fd, buffer)` for immediate writes (D-02 says no buffering). Do NOT use appendFileSync (opens/closes fd each time). Fire-and-forget is NOT acceptable here -- D-02 explicitly requires each event to be on disk. Use writeSync which is synchronous but operates on an already-open fd, so the only cost is the actual I/O.

    **Reverse scan (D-48):**
    1. `fs.statSync(path)` to get file size
    2. Open fd for read
    3. Seek to `fileSize - 4`, read 4B total_len of last event
    4. Seek backward by total_len, read 1B type
    5. If type === 0x02 (SNAPSHOT): read full event, return payload
    6. Else: continue backward using the trailer of the current event
    7. Stop at HEADER_SIZE (6 bytes) if no snapshot found, return null

    **Rotation (D-15, D-49):**
    1. Close current fd
    2. Determine next sequence number by scanning existing `.NNN.bin.gz` files
    3. `gzip events.bin -> events.NNN.bin.gz` using `zlib.createGzip()` pipe
    4. Create new `events.bin` with header
    5. Write METADATA event (D-23)
    6. Write current SNAPSHOT immediately (D-49: ensures self-contained)
    7. Open new fd for writing
    8. Reset event counter

    **D-47:** Snapshots do NOT clear history events. All events are permanently retained.

    **CJS/ESM import note:** This file does NOT import @xterm/headless. It only deals with binary I/O. The headless terminal lives in terminal.ts.
