**Step 4: Create `apps/proxy/src/pty-manager.ts`**

This is the core module. Implement a `PtyManager` class with the following contract:

```typescript
import * as pty from "node-pty";
import type { IPty } from "node-pty";
import type { DataTap } from "./tap.js";
```

**PtyManagerOptions interface:**
```typescript
export interface PtyManagerOptions {
  claudeArgs: string[];       // process.argv.slice(2), per D-02
  tap: DataTap;               // per D-12
  stdin: NodeJS.ReadStream;   // injectable for testing
  stdout: NodeJS.WriteStream; // injectable for testing
}
```

**PtyManager class public API:**
- `constructor(options: PtyManagerOptions)` -- stores options, does NOT spawn yet
- `start(): void` -- spawns claude in PTY, sets up all I/O wiring, raw mode, resize listener
- `cleanup(exitCode: number): void` -- restores raw mode, kills child, exits process

**start() implementation details (all locked decisions applied):**

1. Read terminal dimensions: `this.stdout.columns ?? 80` and `this.stdout.rows ?? 24`
2. Spawn via `pty.spawn("claude", this.claudeArgs, { name: process.env.TERM ?? "xterm-256color", cols, rows, cwd: process.cwd(), env: process.env as Record<string, string> })` (per D-04)
3. If `this.stdin.isTTY === true`, call `this.stdin.setRawMode(true)` then `this.stdin.resume()`. If not TTY, just `this.stdin.resume()` (handles piped input per Pitfall 6)
4. Wire `this.stdin.on("data", (data: Buffer) => { child.write(data.toString()); })` (per D-08, Ctrl+C arrives as \x03)
5. Wire `child.onData((data: string) => { this.stdout.write(data); this.tap(data); })` (per D-11 pure passthrough + D-12 tap)
6. Wire resize with 50ms debounce (per D-09, Pitfall 2):
   ```typescript
   let resizeTimer: ReturnType<typeof setTimeout> | null = null;
   this.stdout.on("resize", () => {
     if (resizeTimer) clearTimeout(resizeTimer);
     resizeTimer = setTimeout(() => {
       child.resize(this.stdout.columns ?? 80, this.stdout.rows ?? 24);
     }, 50);
   });
   ```
7. Wire `child.onExit(({ exitCode, signal }) => { ... })` (per D-05, Pitfall 5):
   - Restore raw mode if interactive
   - Compute exit code: `signal ? 128 + signal : exitCode` (128+signal for signal termination per Pitfall 5)
   - Call `process.exit(code)`
8. Register cleanup handlers (per D-07, Pitfall 1, Pitfall 3):
   - `process.on("SIGTERM", () => this.cleanup(143))`
   - `process.on("SIGHUP", () => this.cleanup(129))`
   - `process.on("uncaughtException", (err) => { process.stderr.write(\`cc-anywhere: fatal error: \${err.message}\n\`); this.cleanup(1); })`
   - `process.on("unhandledRejection", (reason) => { process.stderr.write(\`cc-anywhere: unhandled rejection: \${reason}\n\`); this.cleanup(1); })`
9. Wire `this.stdin.on("end", () => { child.write("\x04"); })` for piped stdin EOF

**cleanup() implementation:**
- If `this.stdin.isTTY`, try `this.stdin.setRawMode(false)` in try/catch
- Try `this.child.kill()` in try/catch (child may already be dead)
- Call `process.exit(exitCode)`

Store `this.child` as a private property set during `start()`.

// 模拟 node-pty 的 IPty 接口
interface MockPty {
  write: ReturnType<typeof vi.fn>;
  resize: ReturnType<typeof vi.fn>;
  kill: ReturnType<typeof vi.fn>;
  onData: ReturnType<typeof vi.fn>;
  onExit: ReturnType<typeof vi.fn>;
  pid: number;
}

let mockPty: MockPty;
let onDataCallback: ((data: string) => void) | null = null;
let onExitCallback: ((e: { exitCode: number; signal?: number }) => void) | null = null;

vi.mock("node-pty", () => ({
  spawn: vi.fn(() => {
    mockPty = {
      write: vi.fn(),
      resize: vi.fn(),
      kill: vi.fn(),
      onData: vi.fn((cb: (data: string) => void) => {
        onDataCallback = cb;
        return { dispose: vi.fn() };

        return { dispose: vi.fn() };

      pid: 12345,

    return mockPty;
  }),
}));
