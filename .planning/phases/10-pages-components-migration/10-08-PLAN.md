<objective>
Close the chat-panel layout critique from Phase 10 UI walkthrough (2026-04-18 session). The bottom input region shows a 5-button vertical SemanticActionPanel, a text-only StatusLine that only ever displays "Claude 正在响应…", and three different icon sizes in one row — visually chaotic and semantically muddled (same button does different things per mode, duplicates ChatHeader overflow, hardcoded with no extension mechanism).

This plan does a **UI-only convergence** (no new features):

1. **StatusLine redesign** — replace 24px text line with a 4px colored bar under ChatHeader, mirroring Feishu's 5-state design (idle/working/waiting_approval/terminated/disconnected) with sweep + breathe animations. Aggregates three previously-scattered signals (connection, approval, working) into one visual channel.

2. **Send button gets Stop variant** — when `isWorking=true`, the paper-plane icon becomes a filled square. Click sends `session_worker_abort` (JSON) or `remote_input_raw: "\x03"` (PTY). This replaces the standalone interrupt button that used to live in SemanticActionPanel.

3. **SemanticActionPanel deleted** — the 5-button vertical column is removed entirely. Its functions either migrate elsewhere or are dropped:
   - interrupt → send-button Stop variant
   - toggle_permission → already in ChatHeader overflow, now works for both modes (proxy side fanout)
   - history_prev / history_next → JSON drops the keyboard shortcut (messages list already shows all history); PTY keeps keyboard ArrowUp/Down behavior
   - cancel → Esc + QuotePreviewBar's own X already cover it

4. **InputBar … menu shell** — add a MoreHorizontal button between textarea and send button, opens a Popover (desktop) or BottomSheet (mobile). Contents are a disabled "more features coming soon" placeholder for now; the shell (anchor, positioning, a11y, responsive switch) is production-ready so the next plan only has to populate menu items.

5. **Permission mode unified across JSON/PTY** — ChatHeader overflow's Permission mode submenu already calls `permission_mode_change` control; proxy handler extended so PTY sessions receive a Tab ANSI write instead of just updating worker env.

Why now: the messy chat panel is the last remaining visual regression from the walkthrough. All other fixes landed in waves 1-7. A single plan of convergence closes it.

Output: 1 rewritten component (StatusLine), 2 new components (SendButton, InputMenu), 1 deletion (SemanticActionPanel + associated ansi-keys exports), 1 new shared schema type, 1 new proxy handler (session_worker_abort) + 1 extended proxy handler (permission_mode_change PTY fanout), updated wiring across chat.tsx / chat-json-view / chat-pty-view / input-bar.
</objective>

<threat_model>
- **Risk A**: Removing SemanticActionPanel breaks PTY-mode control paths (Tab for permission, Ctrl+C for interrupt) that no longer have any entry point. Mitigation: permission mode stays accessible via ChatHeader overflow (already wired + now works for PTY via proxy fanout); interrupt moves to Stop button. Esc / cancel functionality is covered by existing textarea Esc handler and QuotePreviewBar's X.
- **Risk B**: StatusLine state aggregation priority wrong — e.g., showing "working" while disconnected. Mitigation: priority order fixed as `disconnected > waiting_approval > terminated > working > idle`, matching Feishu `index.tsx:375-382`. Unit test covers the priority matrix.
- **Risk C**: Stop button click during a race (isWorking flipped to false between render and click) sends abort to an already-idle session. Proxy must tolerate: if worker already exited or turn completed, abort is a no-op logged at info level, not an error.
- **Risk D**: `session_worker_abort` SIGINT goes to wrong pid if worker was reaped and pid recycled. Mitigation: proxy handler checks `sessionManager.getSession(sid)` exists + `session.state !== TERMINATED` before killing; otherwise logs and drops.
- **Risk E**: InputMenu Popover anchor shift on textarea autosize (1→8 rows) causes visual drift. Mitigation: anchor on the `…` button itself, not the textarea; button sits in a fixed-position flex cell.
- **Risk F**: 4px color bar interferes with touch scroll on mobile (thin horizontal target near header). Mitigation: `pointer-events: none` on the bar (it's purely decorative), touch events pass through to messages area.
- **Risk G**: PTY-side permission mode Tab ANSI write could collide with user input if the Tab arrives mid-keystroke. Mitigation: user-initiated writes to PTY serialize through the same IPC socket; no concurrent writes from proxy side. Log the permission-mode Tab injection for debugging.
- **Risk H**: Removing `SemanticAction` / `ACTION_MAP` / `sendSemanticAction` exports breaks downstream imports. Mitigation: grep for all usages before delete; only `semantic-action-panel.tsx` consumes them (same file being deleted). `sendRemoteInputRaw` stays for send-button Stop path.
</threat_model>

### Task 3 — Shared schema + proxy session_worker_abort handler
**Goal**: Add `session_worker_abort` control message type and implement SIGINT on worker pid.

**Implementation**:
- `packages/shared/src/schemas/relay-control.ts`:
  - Add to the discriminatedUnion: `z.object({ type: z.literal("session_worker_abort"), sessionId: z.string() })`
- `apps/proxy/src/serve.ts`:
  - Add handler branch next to existing `session_terminate`:
    ```ts
    } else if (parsed.type === "session_worker_abort") {
      const sid = parsed.sessionId as string;
      const session = sessionManager.getSession(sid);
      if (!session) {
        logger.warn({ sessionId: sid }, "session_worker_abort: session not found");
      } else if (session.state === SessionState.TERMINATED) {
        logger.info({ sessionId: sid }, "session_worker_abort: already terminated, dropping");
      } else {
        try {
          process.kill(session.pid, "SIGINT");
          logger.info({ sessionId: sid, pid: session.pid }, "session_worker_abort: SIGINT sent to worker");
        } catch (err) {
          logger.warn({ sessionId: sid, pid: session.pid, error: String(err) }, "session_worker_abort: kill failed");
        }
      }
    }
    ```
- No worker-side changes needed: Claude CLI receives SIGINT and aborts its current turn naturally

**Acceptance**:
- Schema validates control message round-trip
- Proxy SIGINT reaches worker pid; worker pid survives (SIGINT only aborts current stream-json turn, doesn't kill the process)
- Invalid sid or terminated session → logged drop, no throw

**Verify**: Unit test the handler with a mocked sessionManager; integration verify via Task 2's Stop button.
