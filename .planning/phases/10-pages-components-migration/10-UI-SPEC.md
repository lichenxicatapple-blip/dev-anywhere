## Spacing Scale

Declared values (Tailwind v4 defaults, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px  (`p-1`, `gap-1`) | Icon gaps, inline tight padding, dot indicator offsets |
| sm | 8px  (`p-2`, `gap-2`) | Compact element spacing, button internal gap with icon |
| md | 12px (`p-3`, `gap-3`) | **Default** element padding (container rows, card internal), matches Phase 7 D-08 "compact tool" feel |
| base | 16px (`p-4`, `gap-4`) | Default section padding, InputBar wrapper, message bubble inner spacing |
| lg | 24px (`p-6`, `gap-6`) | Section separators, empty-state vertical rhythm |
| xl | 32px (`p-8`, `gap-8`) | Major section breaks, onboarding block spacing |
| 2xl | 48px (`p-12`, `gap-12`) | Page-level vertical rhythm (ProxySelect hero spacing on mobile) |

Layout constants (non-uniform — each has a purpose):
- Desktop sidebar width: **280px** fixed (between CONTEXT D-07 260–300 range); collapsed sidebar = **0** (fully hidden) + 40px floating "expand" affordance.
- App chrome header height: **48px** (matches existing `h-12` used in pty-test.tsx line 156).
- InputBar min-height: **48px**; max-height: **240px** (8 × 24px line-height + 48px padding+chrome per D-26).
- Touch targets (mobile session list rows, dropdown items): **44px min height** (iOS HIG); desktop equivalents may shrink to 36px (`h-9`).
- ToolApproval card (PTY mode floating): max-width 360px, bottom-right offset 16px from PTY container edge.
- Sidebar session-list row: 44px height (mobile) / 36px (desktop), 12px horizontal padding.

Exceptions: touch-target overrides above (44px) are the only non-multiple-of-4 deviations; accepted for iOS HIG compliance.

## Typography

Four sizes, two weights. Sans for UI chrome/body, mono for code/technical identifiers/PTY.

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Display (ProxySelect hero, empty-state heading) | 24px (`text-2xl`) | 600 semibold | 1.2 | sans |
| Heading (section titles, Dialog title, session list group header) | 18px (`text-lg`) | 600 semibold | 1.3 | sans |
| Body (message text, settings descriptions, input text) | 14px (`text-sm`) | 400 regular | 1.5 | sans |
| Label / metadata (timestamps, counters, badges, tooltip, status line) | 12px (`text-xs`) | 400 regular | 1.4 | sans |

Mono overrides (**required** — no fallback to sans for these):
- xterm.js PTY content — 14px, 400, line-height locked by xterm renderer (do not override via CSS)
- Fenced code blocks in Markdown — 13px, 400, line-height 1.5, background = `--muted`
- Inline `code` within Markdown — 0.9em relative, 400, background = `--muted`, padding 2px 4px, radius = `--radius-sm`
- Session ID / tool names in status chrome / slash command menu — 12px mono, 400
- File paths in quote-preview-bar and file picker results — 13px mono, 400

Weight contract: **only `400` (regular) and `600` (semibold) are permitted**. No `500`, `700`, or `300`. Button label weight is 500 by default in shadcn — override to 400 to match contract (buttons get emphasis from the amber accent background, not from weight).

Tabular numerals: use `font-variant-numeric: tabular-nums` on timestamps and numeric counters so they don't reflow during updates.

Status colors (not part of 60/30/10 — reserved semantic use only, non-interactive):

| Token | Value | Usage |
|-------|-------|-------|
| `--color-status-working` | `#4FC1FF` cyan | In-progress indicator (spinner, streaming-in-progress dot) |
| `--color-status-success` | `#00D4AA` teal | Connection online dot, completed tool call checkmark |
| `--color-status-warning` | `#E8AB5A` | Reconnecting banner, permission-mode=plan indicator |
| `--color-status-error` | `#F44747` | Error toast, disconnected state |

**Accent (`#D4A574` amber) reserved EXCLUSIVELY for:**
1. Primary button background (Send, Create, Allow, Save)
2. Focus ring (`--ring`) on interactive elements — via shadcn default `focus-visible:ring-ring/50`
3. Selected session row in sidebar (left-edge 2px bar + 8% tint background)
4. Active tab indicator (when split-pane is active on desktop)
5. Progress bars (streaming message progress, PTY reconnection progress)
6. Slash-command and Command Palette selected-item highlight
7. Active link color in Markdown (hover state only)

Accent is **NOT** used for: body text, destructive actions, icons on their own, borders, status dots, hover backgrounds (those use secondary `#2D2D2D`).

**xterm.js theme exception (CONTEXT D-02):** The PTY terminal's `cursorAccent: #00D4AA` stays teal — it's a Phase 9 locked decision. This is the only place teal appears in the UI. Do not change `xtermTheme.cursorAccent`.

**Destructive (`#F44747`) reserved for:**
- Terminate session button (session list item menu, Chat header menu)
- Deny button in ToolApprovalCard
- Destructive confirmation dialog primary action
- Error toast background-left accent strip (not full fill)

## Interaction & Motion

| Token | Value | Usage |
|-------|-------|-------|
| Duration-fast | 120ms | Hover, focus, button press feedback |
| Duration-base | 200ms | Sidebar collapse/expand, popover open/close, dialog fade |
| Duration-slow | 320ms | Page route transitions (mobile only), split-pane emergence |
| Easing-standard | `cubic-bezier(0.2, 0, 0.2, 1)` | Default for all timed transitions (ease-out bias) |
| Easing-accelerate | `cubic-bezier(0.4, 0, 1, 1)` | Exit transitions (popover close, dismissed toast) |

No parallax, no bounce, no springy easing. Developer-tool discipline.

Component states (must all be designed, not just default):

| Component | default | hover | active / pressed | focus-visible | disabled | selected |
|-----------|---------|-------|------------------|---------------|----------|----------|
| Button primary | amber bg | amber/90 | amber/80 | amber ring 3px | 50% opacity + pointer-none | n/a |
| Button ghost | transparent | `--accent` bg | `--accent` bg + 90% | amber ring | 50% opacity | n/a |
| Button destructive | red bg | red/90 | red/80 | red ring | 50% opacity | n/a |
| Session list row | transparent | `--accent` bg | same as hover | amber ring inset | 50% opacity | amber 2px left bar + amber/8 bg + text `--foreground` |
| InputBar textarea | `--input` bg | (no change) | (no change) | amber ring 3px offset 0 | 50% opacity | n/a |
| ToolApproval card | `--card` bg, `--border` 1px | (no change) | (no change) | amber ring on focused button | n/a | n/a |
| Command palette item | transparent | `--accent` bg | `--accent` bg | (keyboard arrow-driven) highlight = amber bg + primary-foreground text | n/a | n/a |

Copy rules:
- No period at end of labels or button text. Periods only in full-sentence body copy.
- Error messages state **what went wrong + next action** — never just "出错了".
- Avoid "请" (please) — it softens imperative UX and wastes width. Exceptions: none.
- Time format: relative ("2 分钟前") for <24h, absolute ("4 月 17 日 14:32") for older.
- Numeric thresholds: show exact numbers, don't round ("12 条消息" not "约 10 条").

## Responsive Breakpoints & Layout Rules

Breakpoints (Tailwind default, CONTEXT D-33):

| Breakpoint | Width | Primary behavior |
|------------|-------|------------------|
| (base) | < 640px | Mobile: three-page navigation stack (ProxySelect → SessionList → Chat), InputBar always visible, single column |
| sm | ≥ 640px | Large-phone / landscape — same as base but allow wider message bubbles (max 600px) |
| md | ≥ 768px | **master-detail activation (D-07)** — sidebar (280px) + main. ProxySelect becomes sidebar-top dropdown, not a page |
| lg | ≥ 1024px | **split-pane activation (D-18)** — main region can host two Chat panes side-by-side |
| xl | ≥ 1280px | (no structural change) — wider content max-width in ChatJsonView (720px message column) |

Viewport rules (D-34):
- Use `100dvh` for full-height containers, never `100vh`.
- Use `env(safe-area-inset-*)` in CSS for iOS notch/home-indicator.
- InputBar position on iOS Safari: listen to `visualViewport.resize`, translate InputBar by `(visualViewport.height - window.innerHeight + visualViewport.offsetTop)` so it sits above the keyboard without JS polyfills for 100vh.

Responsive component behavior:
- `ProxySwitcher`: `layout="page"` on `< md`, `layout="dropdown"` on `≥ md`.
- `SessionList`: `layout="page"` on `< md`, `layout="sidebar"` on `≥ md`.
- `ChatHeader` back button: visible on `< md`, hidden on `≥ md` (replaced by sidebar toggle).
- Split-pane button: disabled and hidden on `< lg`.

## Accessibility Baseline (D-36, D-37)

Required per component, enforced at plan-level checkpoint:

1. Semantic HTML elements: `<nav>` for sidebar, `<main>` for content region, `<header>` for app chrome, `<form>` wrapping InputBar, `<article>` or `<li>` for message items, `<section>` for split-pane columns.
2. Icon-only buttons MUST carry `aria-label` — no exceptions. lucide-react icons are decorative (`aria-hidden="true"`).
3. Focus rings always visible (never `outline: none` without a replacement) — shadcn default `focus-visible:ring-ring/50` suffices.
4. Tab order: logical left-to-right, top-to-bottom. InputBar is last tab stop in Chat so keyboard users can reach it from anywhere with a single wraparound.
5. ToolApproval keyboard: `y/n/a` only active when the card (or its buttons) has focus, to avoid hijacking global typing in InputBar.
6. Command palette: `Esc` closes, `↑↓` navigates, `Enter` activates, type-to-filter.
7. Message list: `aria-live="polite"` on the assistant streaming region so screen readers announce new content without spam.
8. Color contrast: all body text on `--background` ≥ 7:1 (WCAG AAA); `--muted-foreground` on `--background` ≥ 4.5:1 (AA). Confirm via tooling in Plan 10-01 polish pass.
9. Motion: respect `prefers-reduced-motion: reduce` — disable sidebar collapse animation, Dialog fade, and streaming cursor blink.
