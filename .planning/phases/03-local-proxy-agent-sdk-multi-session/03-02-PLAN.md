    SessionManager CRUD:
    - createSession("pty") returns SessionInfo with unique nanoid id, mode "pty", state "idle", createdAt timestamp
    - createSession("json") returns SessionInfo with unique nanoid id, mode "json", state "idle", createdAt timestamp
    - createSession with optional name stores it in SessionInfo
    - listSessions() returns all sessions sorted by createdAt descending
    - getSession(id) returns SessionInfo or undefined
    - getSession(nonexistent) returns undefined
    - terminateSession(id) for a pty session marks state as "terminated"
    - terminateSession(id) for a json session marks state as "terminated" and returns the pid for caller to kill
    - terminateSession(nonexistent) returns false
    - terminateAll() marks all non-terminated sessions as "terminated" and returns json session pids
    - updateState(id, newState) transitions state and triggers persistence save

    State machine (per D-16):
    - idle -> working succeeds
    - working -> waiting_approval succeeds
    - waiting_approval -> idle succeeds (approval cycle)
    - working -> idle succeeds
    - any state -> error succeeds
    - any state -> terminated succeeds
    - terminated -> any state fails (terminal state)
    - error -> idle fails (error is recoverable only via terminated)

    Persistence (per D-17):
    - save() writes session map to JSON file at configurable path using atomic write (write temp, rename)
    - load() reads JSON file and restores session map
    - After createSession, JSON file is updated
    - After terminateSession, JSON file is updated
    - load() with corrupt file throws explicit error (no silent fallback)
    - load() with missing file starts with empty map (fresh start)

    Heartbeat (per D-26, D-27, D-28):
    - recordHeartbeat(sessionId) updates lastHeartbeat timestamp
    - getStaleSessionIds(thresholdMs) returns PTY session IDs where lastHeartbeat is older than threshold
    - Default threshold: 30000ms (3 missed 10-second heartbeats per D-27)

    Reaper (per D-22):
    - startReaper(intervalMs) starts a setInterval that checks JSON sessions
    - Reaper calls isProcessAlive(pid) for each JSON session with a pid
    - If process is dead, reaper marks session as terminated
    - stopReaper() clears the interval
  </behavior>

    - `isProcessAlive(pid)`: try `process.kill(pid, 0)`, return true. catch -> return false.

    Heartbeat stale detection (per D-27):

    ```typescript
    // stream-json 事件类型定义，基于 cc-connect 验证的结构
    export type StreamJsonEventType = "system" | "assistant" | "user" | "result" | "control_request" | "control_cancel_request" | "stream_event";

    export interface StreamJsonEvent {
      type: StreamJsonEventType;
      [key: string]: unknown;
    }

    export type ApprovalStrategy = (
      toolName: string,
      input: Record<string, unknown>,
    ) => Promise<{ behavior: "allow" | "deny"; message?: string }>;

    export interface JsonSessionOptions {
      workDir?: string;          // cwd for claude process, defaults to process.cwd()
      claudeArgs?: string[];     // extra args beyond the required stream-json flags
      approvalStrategy?: ApprovalStrategy;  // per D-21, injectable
      onEvent?: (event: StreamJsonEvent) => void;
      onExit?: (code: number) => void;
    }
    ```

    Default approval strategy (per D-19):
    ```typescript
    const denyAllStrategy: ApprovalStrategy = async (_toolName, _input) => ({
      behavior: "deny" as const,
    ```

    Implement `JsonSession` class:

    **Fields:**
    - `private child: ChildProcess | null = null`
    - `private stderrChunks: string[] = []`
    - `private writeQueue: Promise<void> = Promise.resolve()` -- per Pitfall 7, serializes stdin writes
    - `private readonly options: Required<Pick<JsonSessionOptions, 'workDir' | 'approvalStrategy'>> & JsonSessionOptions`

    **`start(): number`:**
    - Build args: `["--output-format", "stream-json", "--input-format", "stream-json", "--permission-prompt-tool", "stdio", "--verbose", ...extraArgs]`
    - Filter env: `Object.fromEntries(Object.entries(process.env).filter(([k]) => !k.startsWith("CLAUDECODE")))` per Pitfall 4
    - `spawn("claude", args, { cwd: workDir, stdio: ["pipe", "pipe", "pipe"], env: filteredEnv })`
    - Set up stdout parsing: pipe `child.stdout` through `new LineBuffer()`, listen for `"data"` events, parse each line with `JSON.parse`, skip failures (non-JSON verbose output per D-07)
    - For each parsed event: if `type === "control_request"` and `request.subtype === "can_use_tool"`, handle tool approval. Otherwise, call `this.options.onEvent?.(event)`.
    - Set up stderr collection: `child.stderr.on("data", chunk => this.stderrChunks.push(chunk.toString()))`
    - Set up exit handler: `child.on("exit", code => this.options.onExit?.(code ?? 1))`
    - Return `child.pid!`

    **`private async handleControlRequest(event: StreamJsonEvent): Promise<void>`:**
    - Extract `request_id` from `event.request_id` (top-level field, NOT inside `event.request`)
    - Extract `tool_name` and `input` from `event.request`
    - Call `this.options.approvalStrategy(tool_name, input)`
    - Build response:
      - For deny: `{ type: "control_response", response: { subtype: "success", request_id, response: { behavior: "deny", message } } }`
      - For allow: `{ type: "control_response", response: { subtype: "success", request_id, response: { behavior: "allow", updatedInput: {} } } }`
    - Write to stdin via `this.writeToStdin(JSON.stringify(response))`

    **`sendMessage(content: string): void`:**
    - Build message: `{ type: "user", message: { role: "user", content } }`
    - Call `this.writeToStdin(JSON.stringify(message))`

    **`private writeToStdin(data: string): void`:**
    - Append to write queue to prevent interleaved writes (per Pitfall 7):
    ```typescript
    this.writeQueue = this.writeQueue.then(() => {
      return new Promise<void>((resolve, reject) => {
        if (!this.child?.stdin?.writable) { reject(new Error("stdin not writable")); return; }
        this.child.stdin.write(data + "\n", (err) => { err ? reject(err) : resolve(); });
      });
    });
    ```

    **`async stop(gracePeriodMs = 5000): Promise<void>`:**
    - Per RESEARCH "Session Termination with Grace Period":
    - If `!this.child` or child already exited, return
    - Send SIGTERM
    - Poll `isAlive()` every 200ms until dead or grace period expires
    - If still alive after grace period, send SIGKILL

    **`isAlive(): boolean`:**
    - If no child or no pid, return false
    - Try `process.kill(child.pid, 0)`, return true. Catch -> false.

    **`getStderr(): string`:**
    - Return `this.stderrChunks.join("")`
