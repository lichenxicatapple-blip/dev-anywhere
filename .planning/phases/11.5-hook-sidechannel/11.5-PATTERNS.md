## Pattern Assignments

### `apps/proxy/src/serve/hook-approval-manager.ts` (new — service, event-driven)

**Analog:** `apps/proxy/src/serve/tool-approval-manager.ts` (整文件 54 行)

两者均是"纯数据管理器，不做 IO"——唯一差别是 resolver 类型（Hook 侧是 `(decision) => void` 回调，Tool 侧是 `PendingApproval` 元数据）。必须按相同结构组织。

**Imports pattern** (analog 第 1 行):
```typescript
import { serviceLogger } from "../common/logger.js";
```

**Class skeleton pattern** (analog 13–53 行):
```typescript
export class ToolApprovalManager {
  private pending = new Map<string, PendingApproval>();

  register(requestId: string, approval: PendingApproval): void {
    this.pending.set(requestId, approval);
  }

  take(requestId: string): PendingApproval | null {
    const entry = this.pending.get(requestId);
    if (!entry) return null;
    this.pending.delete(requestId);
    return entry;
  }

  cleanupSession(sessionId: string, reason: string): void {
    for (const [requestId, pending] of this.pending) {
      if (pending.sessionId !== sessionId) continue;
      this.pending.delete(requestId);
      serviceLogger.info({ sessionId, requestId, reason }, "Pending tool approval dropped");
    }
  }

  listSession(sessionId: string): Array<...> { ... }
}
```

**Hook 侧必须增加的内容**（在 RESEARCH.md § Pattern: hook-approval-manager 已写完整，对应源结构）：
- `register` 第三参为 `Resolver = (decision: 'allow'|'deny') => void`，内部启动 `setTimeout(APPROVAL_TIMEOUT_MS = 24h)` 自动 auto-deny 并记到 `entry.timer`
- `resolve(requestId, decision) → bool`：存在则 `clearTimeout + entry.resolve(decision) + delete`
- `drop(requestId, reason)`：`req.on("close")` 场景用，**不** resolve（response 已断），只 clearTimeout + delete + log
- `drainSession / drainAll`：按 tool 侧 `cleanupSession` 模式写，但要 resolve("deny") 因为 HTTP response 仍挂住

**禁止复用 ToolApprovalManager**（D-12）：namespace + lifecycle 完全不同。

### `apps/proxy/src/serve/hook-endpoint.ts` (new — service, request-response + async-hold)

**Analog:** 无同类 HTTP server。最近的 transport-level 模块是 `apps/proxy/src/serve/relay-connection.ts`（WebSocket 长连接生命周期 + EventEmitter + options bag）。HTTP 用法直接参考 Node built-in。完整实现框架已在 RESEARCH.md § Pattern: Hook endpoint 贴出，planner 照抄。

**Lifecycle pattern 参考** (relay-connection.ts 95–125 行):
```typescript
constructor(relayUrl: string, options?: RelayConnectionOptions) {
  super();
  this.relayUrl = relayUrl;
  this.proxyId = this.loadOrCreateProxyId(options?.proxyIdPath ?? DEFAULT_PROXY_ID_PATH);
  ...
}
connect(): void { ... }
close(): void { ... }
```

Hook endpoint 不用 class，直接导出 `startHookEndpoint(deps): Promise<{ port, close }>`（RESEARCH.md § Pattern 定了）——因为 HTTP server 的状态足够简单，不需要 EventEmitter + FSM。

**关键点（强制遵守）**：
- `server.setTimeout(0)` / `keepAliveTimeout = 0` / `requestTimeout = 0` 三件套，否则 24h 挂不住（见 Pitfall 3）
- 用 `new URL(req.url, "http://127.0.0.1")` 解析 query，别手拼 regex
- body 累积到 `end` 事件，不用 stream consumer；hook payload 很小
- **解析失败不抛**，log warn + `res.writeHead(200).end("ok")`，D-14 要求绝不拖慢 claude
- envelope 构造走 `buildMessage(type, sessionId, Date.now(), payload, "proxy")`（与 serve.ts:44 同模式）
- PermissionRequest 分支注册 `deps.approvalManager.register(requestId, sessionId, resolver)`，resolver 里调 `res.writeHead(200, {...}).end(responseBody)`；然后 `req.on("close", () => approvalManager.drop(requestId, "request closed"))`

### `apps/proxy/src/common/daemon-id.ts` (new — utility, load-or-create)

**Analog:** `apps/proxy/src/serve/relay-connection.ts` 103–119 行 `loadOrCreateProxyId` (整函数复制改名)

**Full analog excerpt**:
```typescript
private loadOrCreateProxyId(idPath: string): string {
  if (existsSync(idPath)) {
    const existing = readFileSync(idPath, "utf-8").trim();
    if (existing.length > 0) {
      return existing;
    }
  }

  const id = nanoid(21);
  const dir = dirname(idPath);
  if (!existsSync(dir)) {
    mkdirSync(dir, { recursive: true });
  }
  writeFileSync(idPath, id, "utf-8");
  return id;
}
```

Hook 侧差异：
- 改成顶层 `export function loadOrCreateDaemonId(path = DAEMON_ID_PATH): string`（不需 class，复用跨模块）
- `nanoid(21)` 改 `nanoid(12)`（CONTEXT.md specifics 区：daemonId 12 位够用）
- `DAEMON_ID_PATH` 加到 `common/paths.ts`，与 `PID_PATH / SESSIONS_PATH` 同区（paths.ts 18–37 行那一块）

private onHookPermissionDecision(msg: Record<string, unknown>): void {
