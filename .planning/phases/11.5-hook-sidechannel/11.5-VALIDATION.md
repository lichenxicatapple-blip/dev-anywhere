# Phase 11.5 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution. Derived from `11.5-RESEARCH.md` (Validation Architecture section) and CONTEXT.md decisions.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest 2.x (existing in apps/proxy, apps/relay, packages/shared) |
| **Config file** | `apps/proxy/vitest.config.ts` (existing) |
| **Quick run command** | `pnpm --filter @cc-anywhere/proxy run test -- --run <file>` |
| **Full suite command** | `pnpm -r run test -- --run` |
| **Estimated runtime** | ~45s (full) / ~3s (single file) |

Executors must `--run` to avoid watch mode (see RESEARCH pitfall: watch-mode masks failures in CI/agent runs).

---

## Sampling Rate

- **After every task commit:** Run targeted single-file test via `--run <file>`
- **After every plan wave:** Run `pnpm --filter @cc-anywhere/proxy run test -- --run` + `pnpm --filter shared run build` when shared schema changed
- **Before `/gsd-verify-work`:** Full-repo `pnpm -r run test -- --run` + `pnpm -r run typecheck` must be green
- **Max feedback latency:** 60 seconds
