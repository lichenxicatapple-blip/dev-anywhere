<decisions>
## Implementation Decisions

### 架构原则

- **D-01**: **单向数据流**：xterm buffer 是唯一 source of truth；scrollbar 读 xterm 状态、用户手势写 xterm 状态；**浏览器原生滚动不参与**。
- **D-02**: **容器锁死**：xterm mount 点 `height: 100%; overflow: hidden`；任何内部 scroll 都由 xterm 自己管。删除 spacer / sticky 的虚拟滚动方案。
- **D-03**: **scrollbar 显示策略**：默认常显（方便桌面端，移动端也能看到位置）；低频滚动时淡出，频繁滚动时完全显示。具体动效在实现时确定，低优先级。

### safeFit 设计

- **D-04**: `wasAtBottom` 定义：`buffer.active.viewportY >= buffer.active.baseY`（完全对齐 LinkShell `build-terminal-html.mjs:142-152`）。
- **D-05**: safeFit 调用点（所有可能改变视口/buffer 的操作前记录，后按需恢复）：
  1. `terminal_resize` envelope 到达（proxy 推送新 cols/rows）
  2. fontSize 变化（用户缩放）
  3. snapshot apply（重连补全量）
  4. 容器尺寸变化（键盘弹出收起 / 横竖屏 / 分列切换）
- **D-06**: 实现方式：提供 `safeFit(term, op: () => void)` 包装函数，内部做 `was = isAtBottom(); op(); if (was) term.scrollToBottom()`。不要强制使用者每处手写。

### 自绘 scrollbar 设计

- **D-07**: **thumb 位置计算**：`thumbTop = viewportY / (buffer.length - rows) * trackHeight`；`thumbHeight = max(MIN_THUMB, rows / buffer.length * trackHeight)`。订阅 `term.onScroll` 重算。
- **D-08**: **手势**：`onTouchStart` 记起点，`onTouchMove` 算偏移 → `term.scrollToLine(targetLine)`；`onTouchEnd` 无惯性（惯性交给 xterm 内建的 `scrollToLine` 已足够流畅）。
- **D-09**: **桌面鼠标滚轮**：转发到 xterm，不劫持（xterm 内建 wheel 事件已处理）。scrollbar 只响应手势拖动，不响应滚轮。
- **D-10**: **不做 fling/bounce**：iOS momentum 从容器层解藕掉（`overflow: hidden`），不还原那种"滑到底继续弹"的 iOS 感。简化换稳定。

### 清理项

- **D-11**: 删除 `pendingNewFrameRef` 及相关新帧跳过逻辑（双向同步时的创可贴）。
- **D-12**: 删除 spacer div、sticky 定位、subpixel translate 那一整套 173-315 行。
- **D-13**: 删除 viewport padding 的 `updateDynamicPadding` 样逻辑（如果存在）。safe-area 走另一套，不和滚动纠缠。

### 兼容 / 风险

- **D-14**: **PWA 安装态的 safe-area**：`env(safe-area-inset-top/bottom)` 继续用 CSS 变量；容器外层处理 safe-area，xterm 容器不感知。
- **D-15**: **拆迁风险**：重构后可能出现新的"滚动不跟手""键盘弹出不回底""字号变化后歪"之类 regression。对策：
  - 单元测试不现实（xterm + DOM 交互难 mock）
  - 写 e2e（Playwright 手势模拟）覆盖主要场景
  - 真机清单（iOS Safari / iPhone + Android Chrome）手动回归
- **D-16**: **代码量净减**：预期拆掉 ~140 行 hack + 新增 ~100 行自绘组件 → 净减约 40 行。如果实现后净增加，说明设计走偏了，review 时 flag。

</decisions>
