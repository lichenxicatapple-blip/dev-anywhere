# Phase 6 -- UI 设计合同

> 飞书小程序核心交互的视觉和交互合同。由 gsd-ui-researcher 生成，gsd-ui-checker 验证通过。

---

## 设计系统

| 属性 | 值 |
|------|-----|
| 工具 | 无（Taro 小程序，shadcn 不适用） |
| 预设 | 不适用 |
| 组件库 | Taro 内置组件（`View`, `Text`, `ScrollView`, `Input`, `Button`） |
| 图标库 | 无（使用 Unicode 符号和 `Image` 组件内联 SVG） |
| 字体 | UI 使用系统默认字体；终端视图使用 `monospace, Menlo, Courier` |

**Taro pxtransform 说明：** `designWidth: 750` 已启用。所有 CSS `px` 值会按 2:1 比例自动转换为响应式单位（750px 设计稿 = 375pt 屏幕宽度）。需要保留精确像素值的地方（终端字号），CSS 中使用大写 `PX` 后缀。本文档中所有数值均为**逻辑点**（375pt 宽度基准），除非另有说明。

---

## 间距体系

所有值为 4 的倍数，单位为逻辑点：

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 图标内间距、状态圆点外边距 |
| sm | 8px | 紧凑元素间距、终端内容内边距、输入框内部间距、聊天气泡内部间距 |
| md | 16px | 区块内边距、页面水平边距、卡片间距、列表项内边距 |
| lg | 24px | 页面垂直内边距、主要区块间距 |
| xl | 32px | 空状态插图外边距 |
| 2xl | 48px | Phase 6 未使用 |
| 3xl | 64px | Phase 6 未使用 |

例外：
- 触摸目标最小高度 44px（Apple HIG 移动端标准）
- 输入栏高度：48px（舒适拇指触摸区域）
- 会话列表项高度：64px（双行内容 + 状态圆点）
- SessionListItem 和 ToolApprovalCard 的垂直内边距使用 12px：紧凑移动端布局中 8px 太挤而 16px 浪费纵向空间

## 字体排版

| 角色 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| 正文 | 14px | 400（常规） | 1.5（21px） | 聊天气泡文字、会话列表副标题、通用内容、工具审批按钮 |
| 标签 | 12px | 400（常规） | 1.4（16.8px） | 时间戳、状态标签、相对时间、元数据、模式标记 |
| 标题 | 16px | 600（半粗） | 1.3（20.8px） | 导航栏标题、会话列表项标题、区块标题 |
| 终端 | 可变（8/10/12/14/16/20 PX） | 400（常规） | 1.4 倍字号 | PTY 终端视图，仅等宽字体 |

**终端字号使用大写 PX** 绕过 Taro pxtransform。D-11 的 6 档字号：8PX, 10PX, 12PX, 14PX, 16PX, 20PX。默认档位：12PX（索引 2）。

字重约束：全程只使用 400 和 600。

UI 字号：恰好 3 种（12px, 14px, 16px）。终端字号不计入 UI 字号数量，因为它是独立的等宽渲染系统，使用大写 PX 绕过转换。

## 页面结构（源自 D-01, D-24）

三个页面，线性 push 导航（无 TabBar）：

### 页面 1：Proxy 选择（`pages/proxy-select/index`）（spike 已验证）
- **整页深色终端主题**：背景 `#1A1A2E`，全页等宽字体，终端氛围
- **品牌打字机 header**：Multi-text 轮播方案，绿色提示符 `>` + 轮播文字，偏置顶
  - 轮播文字："CC Anywhere" <-> "/untethered @anytime"（斜杠命令 + 文件引用风格，呼应 D-28、D-29）
  - 打字速度 70ms/字符，擦除速度 35ms/字符，打完停留 1.5s，擦完停留 0.3s
  - 闪烁光标 `_`，蓝色 `#1890FF`，530ms 闪烁间隔
  - 预留两行最小高度，避免换行时下方内容抖动
- **Proxy 列表**：半透明卡片 `rgba(255,255,255,0.06)`，微弱边框 `rgba(255,255,255,0.08)`
- 每项：proxy 名称（等宽字体）+ 在线/离线状态
- 在线圆点带发光效果：`box-shadow: 0 0 12px rgba(82,196,26,0.5)`
- 导航栏标题："CC Anywhere"，深色背景 `#1A1A2E` + 白色文字（`navigationBarBackgroundColor: "#1A1A2E"`, `navigationBarTextStyle: "white"`），与页面融为一体
- 下拉刷新重新获取 proxy 列表
- Spike 参考：`apps/feishu/src/pages/spike-typewriter/`

### 页面 2：会话列表（`pages/session-list/index`）
- 导航栏标题：proxy 名称（来自已选 proxy）
- 导航栏右侧："+"图标按钮，新建会话
- 两个区块："活跃会话"和"历史会话"（D-37）
- 活跃会话项：标题（D-03 自动摘要）+ 状态圆点 + 模式标记（PTY/JSON）+ 相对时间
- 历史会话项：会话标题 + 相对时间 + "恢复"可操作指示
- JSON 会话左滑显示"终止"按钮（D-06）；PTY 会话无左滑操作

### 页面 3：聊天（`pages/chat/index`）（spike 部分验证）
- 导航栏标题：会话标题（D-03 自动摘要）
- 根据会话类型切换两种渲染模式：
  - **PTY 模式**：终端视口（深色、等宽字体、可滚动）+ 输入栏。Spike 参考：`spike-render/`
  - **JSON 模式**：聊天气泡列表（IM 风格）+ 输入栏
- **气泡入场动画**（spike 已验证）：纯 CSS `@keyframes`，助手气泡 350ms 带 overshoot 弹性，用户气泡 250ms 快速滑入。Spike 参考：`spike-bubble-anim/`
- 输入栏固定在页面底部，键盘弹起时自动上移
- **消息时间戳**：点击气泡展开/收起时间显示（气泡底部），不常驻占空间
- **聊天设置菜单**（spike 已验证）：
  - 触发：输入栏内发送按钮左侧菜单按钮（`···` 中间点图标，72px 圆形，与发送按钮等大）
  - 面板：底部滑出，圆角顶部，半透明遮罩
  - 动画：弹出 slideUp 300ms + 遮罩 fadeIn 200ms，收起 slideDown 250ms + 遮罩 fadeOut 250ms
  - 内容：
    - Permission 模式切换（Default / Auto Accept / Plan，三选一 chip）
    - 字号调节（A-/A+ 按钮，PTY 和 JSON 通用，双指缩放仍保留）
    - 其他会话级设置（后续扩展）
  - Spike 参考：`spike-chat-json/`

## 交互合同

### 自动滚动（D-18）
1. 新内容到达 -> 用户在距底部 50px 以内时，自动滚动到底部
2. 用户向上滚动超过 100px -> 显示"回到底部"按钮，暂停自动滚动
3. 用户点击"回到底部" -> 300ms 平滑滚动到底部，隐藏按钮
4. 用户发送新消息 -> 滚到底部，隐藏按钮，恢复自动滚动

### 滚动方案（spike 已验证，关键决策）
- **禁止使用 ScrollView 组件用于聊天消息列表**：飞书小程序 ScrollView 存在间歇性滚动粘滞 bug，无法通过代码规避
- **使用 View + CSS `overflow-y: auto` + `-webkit-overflow-scrolling: touch`** 替代，滚动完全丝滑
- 自动滚底：通过底部锚点元素或 JS `scrollTop` 实现，不依赖 `scrollIntoView` API
- 注意：PTY 终端视口仍使用 ScrollView（需要双向滚动 `scrollX + scrollY`），如遇同样问题再切换方案

### 文字复制（spike 已验证）
- 使用 CSS `user-select: text` + `-webkit-user-select: text`，支持部分选中复制，与 View overflow 滚动完美共存
- **禁止使用 `Text selectable` 属性**：在 ScrollView 内会拦截首次触摸导致卡顿
- **禁止在滚动容器上挂触摸事件处理函数**（`onTouchStart/Move/End`、`onScroll`）：同样导致卡顿

### 气泡入场动画（spike 已验证）
1. 助手气泡：CSS `@keyframes` -- `translateY(20px) scale(0.92)` -> 60% 处 `translateY(-4px) scale(1.02)` -> `translateY(0) scale(1)`，350ms ease-out
2. 用户气泡：CSS `@keyframes` -- `translateY(16px) scale(0.95)` -> `translateY(0) scale(1)`，250ms ease-out
3. 实现方式：纯 CSS `@keyframes` + `animation` 属性，不需要 `tt.createAnimation` API
4. Spike 参考：`apps/feishu/src/pages/spike-bubble-anim/`（真机验证通过）

### 流式文本（D-17）
1. 收到 `assistant_message` 且 `isPartial: true` -> 将文本增量追加到当前气泡，显示闪烁光标
2. 收到 `assistant_message` 且 `isPartial: false` 或 `result` 事件 -> 移除闪烁光标，标记气泡完成
3. 同一轮对话的后续增量不创建新气泡

### 发送禁用（D-33，仅 JSON 模式）
1. 会话状态变为 `working` -> 禁用发送按钮（置灰），显示"Claude 正在工作..."提示
2. 收到 `result` 事件 -> 启用发送按钮，隐藏工作中提示
3. PTY 会话：发送按钮始终启用（Claude Code 自身处理排队）

### 工具审批流程（D-25）
1. 收到 `tool_use_request` -> 在消息流中渲染 ToolApprovalCard
2. 用户点击"允许" -> 发送 `tool_approve`，卡片过渡为收起的 ToolCallCard 状态
3. 用户点击"全部允许" -> 发送 `tool_approve` + 会话级白名单标记，卡片过渡为收起状态
4. 用户点击"拒绝" -> 发送 `tool_deny`，卡片显示 `#FF4D4F` 色"已拒绝"标记
5. 一旦允许/拒绝，按钮变为不可交互（防止重复点击）

### 会话生命周期
1. App 冷启动 -> 检查 `tt.getStorageSync` 中缓存的 `proxyId` 和 `sessionId`（D-02）
2. 如果缓存值存在且 proxy 在线 -> 直接跳转到聊天页
3. 如果无缓存值或 proxy 离线 -> 显示 proxy 选择页
4. 终止会话（D-06） -> 确认对话框 -> 发送终止命令 -> 从列表移除 -> 如果正在查看则返回上一页

### PTY 字号调节（D-11）
1. 点击 A- -> 字号档位索引减 1，最小为 0
2. 点击 A+ -> 字号档位索引加 1，最大为 5
3. 双指缩放 -> 计算缩放比例，相应偏移档位索引（spike 已验证）
4. 持久化选中的字号档位到 `tt.setStorageSync`

### 斜杠命令补全（D-28）
1. 输入框首位输入 `/` -> 显示命令选择面板
2. 继续输入 -> 过滤命令列表
3. 点击命令 -> 替换输入内容为该命令，关闭面板
4. 清除 `/` 或向前删除超过 `/` -> 关闭面板

### 文件路径补全（D-29）
1. 输入框中输入 `@` -> 显示文件路径选择面板
2. 面板通过 relay 向 proxy 发送 `dir_list_request`
3. 用户浏览目录 -> 每次导航发送新的 `dir_list_request`
4. 点击文件 -> 在光标位置插入 `@filepath`，关闭面板
5. 在搜索框中输入过滤当前目录列表（客户端过滤）
