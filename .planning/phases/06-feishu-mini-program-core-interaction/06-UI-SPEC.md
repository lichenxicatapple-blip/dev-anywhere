# Phase 6 — UI Design Contract

> Visual and interaction contract for Feishu Mini Program core interaction. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Taro mini program, shadcn not applicable) |
| Preset | not applicable |
| Component library | Taro built-in components (`View`, `Text`, `ScrollView`, `Input`, `Button`) |
| Icon library | none (use Unicode symbols and inline SVG via `Image` component) |
| Font | System default for UI; `monospace, Menlo, Courier` for terminal view |

**Taro pxtransform note:** `designWidth: 750` is active. All CSS `px` values are auto-converted to responsive units at 2:1 ratio (750px design = 375pt screen). To preserve exact pixel values (terminal font sizes), use uppercase `PX` suffix in CSS. All values in this spec are in **logical points** (375pt width) unless noted otherwise.

---

## Spacing Scale

Declared values (multiples of 4, in logical points):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, status dot margin |
| sm | 8px | Compact element spacing, terminal content padding, input field internal padding |
| md | 12px | Chat bubble internal padding, list item padding |
| lg | 16px | Section padding, page horizontal margin, card spacing |
| xl | 24px | Page vertical padding, major section gaps |
| 2xl | 32px | Empty state illustration margin |
| 3xl | 48px | Not used in Phase 6 |

Exceptions:
- Touch targets minimum 44px height (Apple HIG for mobile)
- Input bar height: 48px (comfortable thumb target)
- Session list item height: 64px (two-line content with status dot)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 (21px) | Chat bubble text, session list subtitle, general content |
| Label | 12px | 400 (regular) | 1.4 (16.8px) | Timestamps, status labels, relative time, metadata |
| Heading | 16px | 600 (semibold) | 1.3 (20.8px) | Navigation bar title, session list item title, section headers |
| Terminal | Variable (8/10/12/14/16/20 PX) | 400 (regular) | 1.4x font size | PTY terminal view, monospace only |

**Terminal font sizes use uppercase PX** to bypass Taro pxtransform. The 6 tiers from D-11: 8PX, 10PX, 12PX, 14PX, 16PX, 20PX. Default tier: 12PX (index 2).

Font weight constraint: only 400 and 600 used throughout.

## Page Structure (from D-01, D-24)

Three pages, linear push navigation (no TabBar):

### Page 1: Proxy Selection (`pages/proxy-select/index`)
- Full-screen list of available proxies
- Each item: proxy name (hostname or custom) + online/offline status
- Navigation bar title: "CC Anywhere"
- Pull-to-refresh to re-fetch proxy list

### Page 2: Session List (`pages/session-list/index`)
- Navigation bar title: proxy name (from selected proxy)
- Navigation bar right: "+" icon button to create new session
- Two sections: "Active Sessions" and "History Sessions" (D-37)
- Each active session item: title (D-03 auto-summary) + state dot + mode tag (PTY/JSON) + relative time
- Each history session item: session title + relative time + "resume" affordance
- Left-swipe on JSON session reveals "Terminate" button (D-06); PTY sessions have no swipe action

### Page 3: Chat (`pages/chat/index`)
- Navigation bar title: session title (auto-summary from D-03)
- Renders in two modes based on session type:
  - **PTY mode**: Terminal viewport (dark, monospace, scrollable) + input bar
  - **JSON mode**: Chat bubble list (IM-style) + input bar
- Input bar pinned to bottom, above keyboard when active

## Interaction Contracts

### Auto-scroll (D-18)
1. New content arrives -> if user is within 50px of bottom, auto-scroll to bottom
2. User scrolls up more than 100px from bottom -> show "back to bottom" button, pause auto-scroll
3. User taps "back to bottom" -> scroll to bottom with 300ms smooth animation, hide button
4. User sends new message -> scroll to bottom, hide button, resume auto-scroll

### Streaming Text (D-17)
1. `assistant_message` with `isPartial: true` -> append text delta to current bubble, show blinking cursor
2. `assistant_message` with `isPartial: false` or `result` event -> remove blinking cursor, mark bubble complete
3. Never create a new bubble for continuation deltas of the same turn

### Send Disable (D-33, JSON only)
1. Session state becomes `working` -> disable send button (grey), show "Claude is working..." label
2. `result` event received -> enable send button, hide working label
3. PTY sessions: send button always enabled (Claude Code handles its own queuing)

### Tool Approval Flow (D-25)
1. `tool_use_request` arrives -> render ToolApprovalCard in message flow
2. User taps "Allow" -> send `tool_approve`, card transitions to collapsed ToolCallCard state
3. User taps "Allow All" -> send `tool_approve` + flag for session whitelist, card transitions to collapsed state
4. User taps "Deny" -> send `tool_deny`, card shows "Denied" stamp in `#FF4D4F`
5. Once approved/denied, buttons become non-interactive (prevent double-tap)

### Session Lifecycle
1. App cold start -> check `tt.getStorageSync` for last `proxyId` and `sessionId` (D-02)
2. If stored values exist and proxy is online -> navigate directly to chat page
3. If no stored values or proxy offline -> show proxy select page
4. Session terminate (D-06) -> confirmation dialog -> send terminate command -> remove from list -> navigate back if currently viewing

### PTY Font Size (D-11)
1. A- button tap -> decrease font tier index by 1, min 0
2. A+ button tap -> increase font tier index by 1, max 5
3. Pinch gesture -> calculate ratio, offset tier index accordingly (spike-validated)
4. Persist selected font tier to `tt.setStorageSync`

### Slash Command Completion (D-28)
1. Input starts with `/` -> show command picker
2. User continues typing -> filter command list
3. User taps command -> replace input content with command, dismiss picker
4. User clears `/` or deletes back past it -> dismiss picker

### File Path Completion (D-29)
1. Input contains `@` -> show file path picker
2. Picker sends `dir_list_request` via relay to proxy
3. User navigates directories -> each navigation sends new `dir_list_request`
4. User taps file -> insert `@filepath` at cursor position, dismiss picker
5. Typing in search field filters current directory listing client-side
