**New file: `apps/proxy/src/handlers/terminal-push.ts`**

Handles PTY terminal frame push with incremental line-level diff:

```typescript
export interface TerminalPushHandler {
  start(sessionId: string, tracker: TerminalTracker): void;
  stop(sessionId: string): void;
  stopAll(): void;
}

export function createTerminalPushHandler(
  send: (data: string) => void,
  logger: Logger
): TerminalPushHandler
```

Implementation:
1. `FRAME_PUSH_INTERVAL_MS = 200` (5fps)
2. Per-session state: `lastGrid: TermLine[] | null`, interval handle
3. `start()`: set up periodic timer, every 200ms:
   - Call `tracker.hasGridChanged()`, skip if no change
   - Call `tracker.extractGrid()` to get current grid
   - If `lastGrid` is null (first frame): send full grid with `mode: "full"`, all lines in `lines` array
   - If `lastGrid` exists: compare line by line, collect changed lines as `{ lineIndex: number, spans: TermSpan[] }[]`. Send with `mode: "delta"`, only changed entries in `lines` array. If no lines changed (hasGridChanged was false positive from cursor move etc.), skip send.
   - Update `lastGrid` to current grid (deep copy)
   - Build `terminal_frame` MessageEnvelope and call `send()`
4. Also integrate OSC: accept `onOscSignal` callback. In the PtyManager `onData` path, pass raw data through `extractOscSignals()`. If result is non-null and state is not `mid_pause`, build and send `pty_state` envelope.
5. `stop()`: clear interval, remove session state. `stopAll()`: clear all.
