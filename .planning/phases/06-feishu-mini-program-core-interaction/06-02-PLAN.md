```typescript
// OSC 0: 窗口标题 -- ESC ] 0 ; <title> BEL/ST
// OSC 9: 通知 -- ESC ] 9 ; <text> BEL/ST
const OSC_REGEX = /\x1b\](\d+);([^\x07\x1b]*?)(?:\x07|\x1b\\)/g;

export type PtySemanticState = "working" | "turn_complete" | "approval_wait" | "mid_pause";

export interface PtyStateEvent {
  state: PtySemanticState;
  title?: string;
  tool?: string;
}
```

Implement `extractOscSignals(rawData: string): PtyStateEvent | null`:
1. Use `OSC_REGEX` with `exec()` loop to collect all matches as `{ code: number, text: string }` array
2. Look for OSC 9 match first (higher priority per D-39):
   - If text includes `"waiting for your input"` -> return `{ state: "turn_complete" }`
   - If text includes `"needs your permission"` -> return `{ state: "approval_wait", tool: extractedToolName }`. Tool name extraction: match pattern `/permission.*?:\s*(\S+)/` or similar based on Claude Code's actual format.
3. If no OSC 9 but OSC 0 exists -> return `{ state: "mid_pause", title: osc0Text }`
4. If no OSC matches at all -> return `null`

IMPORTANT: Reset regex lastIndex before each call since regex has `g` flag, or use a fresh regex per call to avoid state leakage between calls.

Export `PtySemanticState`, `PtyStateEvent`, and `extractOscSignals`.
