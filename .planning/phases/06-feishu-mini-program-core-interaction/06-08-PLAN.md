- **Mode switching** (D-13):
  - If `currentSessionMode === "pty"`: render StatusLine + TerminalViewport + InputBar
  - If `currentSessionMode === "json"`: render StatusLine + ChatBubbleList + InputBar + BackToBottomButton

- **PTY mode message handling**:
  - Receive `terminal_frame` envelopes -> dispatch to terminal store (SET_TERMINAL_LINES)
  - Receive `pty_state` envelopes -> dispatch to terminal store (SET_PTY_STATE)
  - Send: when user types and sends, build `user_input` envelope with `payload: { text }` and send via relay client. For PTY, this writes to stdin.

- **JSON mode message handling** (D-17, D-19):
  - Receive `assistant_message` envelopes -> parse `payload.text` with `parseAssistantMessage()` -> route with `routeStreamEvent()` -> dispatch to chat store
  - Receive `tool_use_request` envelopes -> add to pending approvals in chat store
  - Receive `tool_result` envelopes -> update tool call output in chat store
  - Receive `session_status` envelopes -> update session state, toggle working indicator
  - Send: build `user_input` envelope and send. Add message to local chat store immediately (optimistic). Clear input, scroll to bottom.

- **D-33 send disable logic**:
  - JSON mode: `disabled = chatState.isWorking || chatState.pendingApprovals.some(a => a.status === "pending")`
  - PTY mode: `disabled = false` (always enabled)
  - `disabledReason`:
    - `isWorking`: "Claude is working..." (UI-SPEC copy)
    - `pendingApprovals.length > 0`: "Waiting for tool approval..." (UI-SPEC copy)

- **Timestamp on tap**: Each bubble shows timestamp when tapped (toggle visibility). Timestamp text at bubble bottom, 12px `#999999`.

- **Keyboard handling**: Input component `adjustPosition` prop for keyboard push-up. On focus, if near bottom, scroll to bottom after keyboard animation.
