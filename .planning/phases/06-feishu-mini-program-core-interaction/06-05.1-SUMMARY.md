**Two-layer message boundary cleanup: Envelope for replayable conversation records, Control for ephemeral push data and infrastructure messages**

## Accomplishments

### Task 1: Shared Schema Refactoring
- Deleted `PtySnapshotPayloadSchema` from session.ts
- Removed `pty_snapshot`, `terminal_frame`, `pty_state` from MessageEnvelopeSchema (18 -> 15 types)
- Added `terminal_frame`, `pty_state`, `terminal_lines_request`, `terminal_lines_response` to RelayControlSchema
- Updated tests: envelope tests verify rejection of removed types, relay-control tests verify new Control types

### Task 2: TerminalTracker lineId Mechanism
- Added `nextLineId` monotonic counter initialized from buffer initial length, incremented via `onLineFeed`
- Implemented `extractLines(fromLineId, count)` with buffer bounds clamping
- Implemented `getOldestLineId()` and `getNewestLineId()` boundary methods
- Extracted `lineToSpans()` private method from inline extractGrid logic for reuse
- Added 6 lineId-specific tests (all passing)

### Task 3: Relay Compression Removal and Control Routing
- Gutted `buffer-compressor.ts` (scheduled for git rm)
- Removed `SNAPSHOT_TYPES` and `compressOnSnapshot` from router.ts
- Session buffer is now append-only
- Added `terminal_frame`, `pty_state`, `terminal_lines_response` to proxy handler's PROXY_TO_CLIENT_TYPES forwarding set
- Added `terminal_lines_request` routing in client handler (forward to bound proxy)
- Updated relay-resilience tests to reflect append-only behavior
- Removed compressOnSnapshot tests from session-buffer.test.ts

### Task 4: Proxy Cleanup and Control Format
- Deleted `sendPtySnapshot` function and its idle-check call from serve.ts
- Gutted `send-pty-snapshot.test.ts` (scheduled for git rm)
- terminal-push.ts now sends Control format `terminal_frame` (no seq/timestamp/source/version)
- Added `handleTerminalLinesRequest` to control-messages.ts with tracker registry
- Added `registerTracker` method for session-tracker association
- Wired `terminal_lines_request` dispatch in serve.ts relay message handler
- Updated terminal-push.test.ts to verify Control format assertions

### Task 5: Integration Tests
- Rewrote `message-routing.test.ts` with 15 comprehensive test cases
- Covers: Envelope routing (assistant_message, tool_approve with whitelistTool)

- Covers: Request-response (dir_list, session_history, terminal_lines full round trip)
- Covers: proxy_list_response with name, buffer behavior verification, error boundaries
