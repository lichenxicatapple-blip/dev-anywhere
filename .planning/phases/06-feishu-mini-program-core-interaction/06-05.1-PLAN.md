<objective>
协议层重构：清理旧 PTY 事件流遗留，建立清晰的两层消息边界，新增 scrollback 按需拉取机制。

背景：cross-AI review 发现 terminal_frame/pty_state 放在 Envelope 层（有 seq、被 buffer、可重放）语义错误。
它们是临时的画面推送，不需要重放。pty_snapshot 是旧事件流方案的遗留物，当前架构下无消费方。
buffer compression 基于 pty_snapshot 丢弃对话消息，实际上在破坏客户端需要的聊天历史。

两层消息的判断标准：断线重连后客户端需要这条消息吗？需要就是 Envelope，不需要就是 Control。

同时新增 terminal_lines_request/response 机制，使用 lineId 寻址，支持手机端双向虚拟滚动浏览 PTY 历史输出。
</objective>

  <action>
在 TerminalTracker 中新增：

```typescript
private nextLineId: number;

// 构造函数中初始化
constructor(...) {
  ...
  this.nextLineId = this.terminal.buffer.active.length;
  this.terminal.onLineFeed(() => {
    this.nextLineId++;
  });
}

getOldestLineId(): number {
  return this.nextLineId - this.terminal.buffer.active.length;
}

getNewestLineId(): number {
  return this.nextLineId - 1;
}

extractLines(fromLineId: number, count: number): TermLine[] {
  const buf = this.terminal.buffer.active;
  const oldestId = this.getOldestLineId();
  const newestId = this.getNewestLineId();

  // 请求范围完全在 buffer 之外
  if (fromLineId > newestId || fromLineId + count <= oldestId) {
    return [];
  }

  // 裁剪到有效范围
  const startId = Math.max(fromLineId, oldestId);
  const endId = Math.min(fromLineId + count, newestId + 1);

  const lines: TermLine[] = [];
  for (let id = startId; id < endId; id++) {
    const bufIdx = id - oldestId;
    const line = buf.getLine(bufIdx);
    if (line) {
      lines.push(this.lineToSpans(line));
    }
  }
  return lines;
}
```

注意：`lineToSpans` 是从已有的 `extractGrid` 中提取的行转换逻辑。如果 extractGrid 里是内联的，先重构为独立的 private 方法再复用。
