    **Create `apps/proxy/src/serve.ts`**

    This is the service daemon entry point (per D-01). It starts a Unix domain socket server, instantiates SessionManager, handles IPC messages from clients, and manages JSON session processes.

    **Constants:**
    ```typescript
    const CC_DIR = `${process.env.HOME}/.cc-anywhere`;
    const SOCK_PATH = `${CC_DIR}/cc-anywhere.sock`;
    const PID_PATH = `${CC_DIR}/cc-anywhere.pid`;
    const PERSIST_PATH = `${CC_DIR}/sessions.json`;
    const LOG_PATH = `${CC_DIR}/service.log`;
    ```

    **Import pino** for structured logging (service has no terminal). Create logger with `pino({ level: "info" }, pino.destination(LOG_PATH))`. Create `CC_DIR` with `mkdirSync` before initializing logger.

    **`export async function startService(): Promise<void>`:**

    1. **Stale socket cleanup (per RESEARCH Pitfall 2):**
       - If `SOCK_PATH` exists, try connecting to it. If connection succeeds, another service is running -- log error and `process.exit(1)`. If connection fails (ECONNREFUSED), socket is stale -- `unlinkSync(SOCK_PATH)`.
       - If `PID_PATH` exists, read PID, check if alive via `process.kill(pid, 0)`. If alive, exit with error. If dead, unlink PID file.

    2. **Create SessionManager:**
       ```typescript
       const sessionManager = new SessionManager({ persistPath: PERSIST_PATH });
       sessionManager.startReaper();
       ```

    3. **Track active JsonSession instances:**
       `const jsonSessions = new Map<string, JsonSession>();`

    4. **Create Unix socket server:**
       ```typescript
       const server = createServer((socket) => {
         handleClientConnection(socket, sessionManager, jsonSessions, logger);
       });
       ```

    5. **Listen on socket, write PID file:**
       ```typescript
       server.listen(SOCK_PATH, () => {
         writeFileSync(PID_PATH, String(process.pid));
         // Set socket permissions to owner-only (per RESEARCH Open Question 1)
         chmodSync(SOCK_PATH, 0o600);
         logger.info({ pid: process.pid, sock: SOCK_PATH }, "Service started");
       });
       ```

    6. **Signal cleanup (per D-23):**
       ```typescript
       async function shutdown() {
         logger.info("Shutting down service");
         sessionManager.stopReaper();
         // terminateAll JSON sessions
         const pids = sessionManager.terminateAll();
         for (const [id, js] of jsonSessions) {
           await js.stop();
         }
         jsonSessions.clear();
         server.close();
         try { unlinkSync(SOCK_PATH); } catch {}
         try { unlinkSync(PID_PATH); } catch {}
         process.exit(0);
       }
       process.on("SIGTERM", shutdown);
       process.on("SIGINT", shutdown);
       ```

    **`function handleClientConnection(socket, sessionManager, jsonSessions, logger):`**

    Set up IPC reader on socket via `createIpcReader(socket, handler)`.

    Handle each IPC message type:

    - **`session_create_request`**: If mode is "pty": `sessionManager.createSession("pty", name)`, reply with `session_create_response { sessionId }`. If mode is "json": `sessionManager.createSession("json", name)`, create new `JsonSession` with `onEvent` that broadcasts to connected clients and `onExit` that marks session terminated. Call `jsonSession.start()`, store PID via `sessionManager.setPid(id, pid)`. Store in `jsonSessions` map. Reply with `session_create_response { sessionId }`.

    - **`session_list_request`**: `sessionManager.listSessions()`, reply with `session_list_response`.

    - **`session_terminate_request`**: `sessionManager.terminateSession(sessionId)`. If json session, call `jsonSessions.get(id)?.stop()` and remove from map. Reply with `session_terminate_response { sessionId, success }`.

    - **`pty_register`**: `sessionManager.updateState(sessionId, "idle")`. This confirms the PTY session is live. Record heartbeat. Log the registration.

    - **`pty_deregister`**: `sessionManager.terminateSession(sessionId)`. Client is closing terminal. Log.

    - **`pty_output`**: Forward to relay when available (Phase 4). For now, log at debug level. This is the tap sideband data from D-12.

    - **`pty_input`**: Write to socket of the client that owns this PTY session. Per D-13, this is for future Feishu input forwarding. For now, just forward if client socket is tracked.

    - **`heartbeat`**: `sessionManager.recordHeartbeat(sessionId)`. Reply with `heartbeat_ack`.

    - **`session_status_update`**: `sessionManager.updateState(sessionId, state)`. This is how clients report state changes (e.g., PTY session started working).

    Track client sockets per session to enable targeted forwarding. Use a `Map<string, net.Socket>` mapping sessionId to client socket.

    Handle socket `"close"` event: find sessions owned by this socket, mark as terminated if they were PTY sessions (the terminal closed without clean deregister).

    Handle socket `"error"` event: log warning, same cleanup as close.

    **Create `apps/proxy/src/client.ts`**

    This is the thin client (per D-02). It connects to the service via Unix domain socket, requests a PTY session, and bridges terminal I/O. PTY runs in the client process for zero latency (per D-04).

    **Constants:**
    ```typescript
    const CC_DIR = `${process.env.HOME}/.cc-anywhere`;
    ```

    **`export async function startClient(claudeArgs: string[]): Promise<void>`:**

    1. **Ensure service is running (per D-03, RESEARCH Pattern 1):**
       ```typescript
       const socket = await ensureService();
       ```

       `async function ensureService(): Promise<net.Socket>`:
       - Try `tryConnect(SOCK_PATH)`. If succeeds, return socket.
       - If fails, auto-start service:
         - Determine path to serve entry: `fileURLToPath(new URL("./serve.js", import.meta.url))` for the compiled serve.js in dist/. But since this is a CLI tool, the entry point is `dist/index.js serve`. So spawn `process.execPath` with args `[path.join(__dirname_equivalent, "index.js"), "serve"]` using `detached: true, stdio: "ignore"`, then `child.unref()`.
         - Alternative simpler approach: spawn `process.execPath` with the resolved path to the serve module directly. The serve module should handle being the main entry point. Export a `main()` from serve.ts that calls `startService()`.
         - Poll with backoff: try connecting up to 20 times with 100ms * (i+1) delay. Return socket on success. Throw after all retries fail.

       `function tryConnect(sockPath: string): Promise<net.Socket | null>`:
       - `return new Promise(resolve => { const s = connect(sockPath); s.on("connect", () => resolve(s)); s.on("error", () => resolve(null)); })`

    2. **Set up IPC reader on socket:**
       ```typescript
       createIpcReader(socket, handleServiceMessage);
       ```

    3. **Request PTY session creation:**
       ```typescript
       socket.write(serializeIpc({ type: "session_create_request", mode: "pty" }));
       ```

       Wait for `session_create_response` (use a one-time promise pattern).

    4. **Create DataTap that forwards PTY output to service (per D-12):**
       ```typescript
       const tap: DataTap = (data: string) => {
         socket.write(serializeIpc({ type: "pty_output", sessionId, data }));
       };
       ```

    5. **Create PtyManager with exit callback (per Plan 01 refactored API):**
       ```typescript
       const ptyManager = new PtyManager({
         claudeArgs,
         tap,
         stdin: process.stdin,
         stdout: process.stdout,
         onSessionExit: (code: number) => {
           // Deregister session with service
           socket.write(serializeIpc({ type: "pty_deregister", sessionId }));
           socket.end();
           process.exit(code);
         },
       });
       ptyManager.start();
       ```

    6. **Register PTY session with service:**
       ```typescript
       socket.write(serializeIpc({ type: "pty_register", sessionId }));
       ```

    7. **Start heartbeat timer (per D-26):**
       ```typescript
       const heartbeatInterval = setInterval(() => {
         socket.write(serializeIpc({ type: "heartbeat", sessionId }));
       }, 10_000);
       ```

    8. **Handle pty_input from service (per D-13, future Feishu input):**
       In `handleServiceMessage`: if message is `pty_input`, write `message.data` to PtyManager via `ptyManager.writeToChild?.(data)`.
       NOTE: PtyManager currently doesn't expose a `writeToChild` method. The stdin piping goes directly from process.stdin. For pty_input injection, we need to add a `write(data: string): void` method to PtyManager that calls `this.child?.write(data)`. This is a small addition to pty-manager.ts -- add it here.

    9. **Signal handling (per D-24, D-25):**
       ```typescript
       function cleanupAndExit(code: number) {
         clearInterval(heartbeatInterval);
         socket.write(serializeIpc({ type: "pty_deregister", sessionId }));
         ptyManager.cleanup(code);
         // onSessionExit callback handles process.exit
       }
       process.on("SIGINT", () => cleanupAndExit(130));
       process.on("SIGTERM", () => cleanupAndExit(143));
       ```
       NOTE: SIGINT should NOT trigger cleanupAndExit by default because Ctrl+C in the terminal should be forwarded to the PTY child (claude), not kill the client. Only register SIGTERM handler. For SIGINT, let the PTY child handle it -- when claude exits from Ctrl+C, the onExit callback fires which triggers deregistration.

    Add a public `write(data: string): void` method that calls `this.child?.write(data)`. This enables pty_input injection from the client.

    ```typescript
    import { Command } from "commander";
    ```
