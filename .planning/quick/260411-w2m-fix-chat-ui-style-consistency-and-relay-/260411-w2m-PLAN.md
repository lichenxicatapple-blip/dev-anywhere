<objective>
Fix two issues blocking usable chat UI in the Feishu mini program:

1. **CSS font sizes are HALF of SPIKE values** — All chat-related components use "actual pixel" values (e.g., `font-size: 14px`) instead of 750-design-width values (e.g., `font-size: 28px`). Taro's pxtransform with `designWidth: 750` converts these further, resulting in unreadably small text (e.g., 5.2px actual). Session list components (.sli-*) are already correct — only chat components need fixing.

2. **Taro H5 WebSocket doesn't connect in Playwright** — `Taro.connectSocket` polyfill produces zero console output and no WS connections, while direct `new WebSocket()` from browser works fine. Needs runtime diagnosis.

Purpose: Make chat UI readable and functional for H5 testing and mini program deployment.
Output: Corrected CSS files matching SPIKE reference values, and working WebSocket connectivity in H5 mode.
</objective>

App initialization: `apps/feishu/src/app.tsx`
Taro H5 WS polyfill: `@tarojs/taro-h5/dist/api/network/websocket/` — wraps native `new WebSocket()`

The conversion rule: Taro pxtransform with designWidth 750 on a 390px screen produces `actual = CSS_value * (390/750) = CSS_value * 0.52`. Current CSS uses "actual pixel" target values but pxtransform still halves them again.
