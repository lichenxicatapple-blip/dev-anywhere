# Quick Task 260413-414: Server-side scrolling

## Summary

Replaced client-side scrollback assembly with server-side scrolling. Client now sends `terminal_scroll_request` with direction/delta, server maintains `viewportOffset` on TerminalTracker, extracts grid at offset from persistedLines + xterm buffer, and pushes a single `terminal_frame` back. Client is a pure frame renderer.

## Changes

### Added
- `terminal_scroll_request` schema in relay-control.ts (direction: up/down, delta: positive int)
- `isScrolled` flag on TerminalFrameFullSchema to distinguish scroll responses from live frames
- TerminalTracker: `viewportOffset`, `scrollUp()`, `scrollDown()`, `resetScroll()`, `extractGridAtOffset()`
- `pty_scroll_request` IPC message type
- Server-side scroll handler in terminal.ts (receives scroll request, extracts grid at offset, pushes frame)
- Scroll routing in serve.ts (terminal_scroll_request -> pty_scroll_request)
- TerminalViewport `onScroll` prop for sending scroll commands
- Scroll request throttling (200ms) in TerminalViewport

### Removed
- `terminal_lines_request` / `terminal_lines_response` protocol (schema, IPC, handlers)
- `ScrollbackCache` service and tests
- All scrollback-related state from terminal-store (scrollbackLines, isLoadingScrollback, isAtOldest, etc.)
- Dead scrollback cache code from TerminalFrameRenderer

### Fixed
- Relay build script: tsbuildinfo path corrected (was deleting wrong file, .d.ts never generated)
