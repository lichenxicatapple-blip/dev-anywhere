<objective>
Server-side scrolling for PTY terminal. Move all scroll logic from client to server.

Currently the client requests historical lines via terminal_lines_request, assembles
scrollback in ScrollbackCache with gap/dedup logic, and manages two data sources
(scrollback + viewport). This refactors to: client sends "scroll up/down N lines"
(terminal_scroll_request), server reads from persistedLines + xterm buffer at offset,
pushes a single terminal_frame back. Client becomes a pure frame renderer.

Purpose: Eliminate complex client-side ScrollbackCache, gap filling, and dedup logic.
Single data source (server-pushed frame) simplifies client and removes an entire class
of synchronization bugs.

Output: Working server-side scroll with clean client.
</objective>
