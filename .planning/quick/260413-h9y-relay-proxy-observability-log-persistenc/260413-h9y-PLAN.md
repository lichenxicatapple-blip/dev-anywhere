**Relay log persistence** (`apps/relay/src/index.ts`):

Replace the plain `pino({ level })` with a pino instance that writes to file. Use `pino.destination()` for file output to `~/.cc-anywhere/logs/relay.log`. In dev mode (`NODE_ENV !== 'production'`), use `pino.multistream` to write to both stdout and file simultaneously. In production mode (Docker), write stdout only (container log collection handles persistence, and ~/.cc-anywhere won't exist in container).

Implementation:
```typescript
import { mkdirSync } from "node:fs";

const LOG_DIR = `${process.env.HOME}/.cc-anywhere/logs`;
const LOG_FILE = `${LOG_DIR}/relay.log`;

let logger: Logger;
if (process.env.NODE_ENV === "production") {
  // Container: stdout only, orchestrator handles log collection
  logger = pino({ level: process.env.LOG_LEVEL ?? "info" });
} else {
  // Dev: dual write to stdout + file for local debugging
  mkdirSync(LOG_DIR, { recursive: true });
  logger = pino(
    { level: process.env.LOG_LEVEL ?? "info" },
    pino.multistream([
      { stream: process.stdout },
      { stream: pino.destination(LOG_FILE) },
    ]),
  );
}
```
