    **1. RelayConnection state machine (relay-connection.ts):**

    Add exported `RelayConnectionState` type:
    ```typescript
    export const RelayConnectionState = {
      DISCONNECTED: "disconnected",
      CONNECTING: "connecting",
      REGISTERING: "registering",
      SYNCED: "synced",
      WAITING_RECONNECT: "waiting_reconnect",
      CLOSED: "closed",
    } as const;
    export type RelayConnectionState = (typeof RelayConnectionState)[keyof typeof RelayConnectionState];
    ```

    Replace `closed: boolean` with `private connectionState: RelayConnectionState = RelayConnectionState.DISCONNECTED`.

    Add `private transition(to: RelayConnectionState): void` that logs the transition and sets state. No from-state validation needed here (relay-connection state transitions are driven by WebSocket events which are inherently valid).

    Update `connect()`: transition to CONNECTING, remove `this.closed = false`.
    Update `doConnect()`:
    - On `ws.open`: transition to REGISTERING (not SYNCED yet, register message sent here).
    - On `proxy_register_response` received: transition to SYNCED, then flush queue.
    - On `ws.close`: if state !== CLOSED, transition to WAITING_RECONNECT and scheduleReconnect.
    - On `ws.error`: log only (close event will follow).

    Update `sendRaw()`:
    - if state === SYNCED: send directly via ws.
    - if state === CLOSED: log warning, discard message (do NOT queue).
    - else: queue the message.

    Update `close()`: transition to CLOSED, clear timer, send proxy_disconnect if ws open, close ws.

    Update `getStatus()`: add `connectionState` field.

    Add queue max capacity to `MemoryMessageQueue` or directly in RelayConnection: cap at 10000 items, drop oldest on overflow with a log warning.

          pending.resolve({ behavior: "deny", message: "Worker disconnected" });
          pendingToolApprovals.delete(requestId);
          logger.info({ sessionId, requestId }, "Pending tool approval denied on worker disconnect");

      }
    });
