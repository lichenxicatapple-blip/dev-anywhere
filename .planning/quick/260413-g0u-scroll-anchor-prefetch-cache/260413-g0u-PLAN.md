scroll interruption during PTY output, and add client-side prefetch cache for historical frames.

when new lines arrive, and every scroll gesture costs a network round-trip with no caching.

Output: Stable scroll anchoring, no output interruption, cached historical frames on client.
</objective>

    TerminalTracker anchor scroll:
    - scrollUp(delta): compute anchorLineId = (newestLineId - rows + 1) - cumulative_offset. Store as private anchorLineId (number | null). null = live mode, number = anchored.
    - scrollDown(delta): move anchorLineId forward. If anchorLineId + rows >= newestLineId, clear anchor (set to null) = back to live.
    - isAnchored(): returns anchorLineId !== null
    - getAnchorLineId(): returns anchorLineId
    - clearAnchor(): sets anchorLineId to null
    - extractGridAtOffset(): when anchored, use anchorLineId as start to extract rows. When not anchored, return extractGrid() (live).
    - Remove viewportOffset, getViewportOffset(), resetScroll() entirely.

    terminal.ts tap callback:
    - When tracker.isAnchored(), do NOT call clearAnchor. Just feed data. The anchor stays put.
    - When not anchored, feed data normally (no resetScroll needed since it's removed).

       - If already anchored, subtract delta from anchorLineId.
       - Clamp to oldestLineId as minimum.
    3. Rewrite `scrollDown(delta)`:
       - If not anchored, no-op (already at live).
       - If anchored, add delta to anchorLineId. If anchorLineId + rows > newestLineId, set anchorLineId = null (back to live).
    4. Add `isAnchored(): boolean` returning `this.anchorLineId !== null`.
    5. Add `getAnchorLineId(): number | null` returning `this.anchorLineId`.

  <action>
    **terminal-store.ts changes:**
    1. Add to TerminalStoreState:
       - `frameCache: Map<number, TermLine[]>` -- keyed by anchorLineId, stores historical frames
       - `anchorLineId: number | null` -- current scroll anchor (null = live mode)
       - `newestLineId: number | null` -- latest known newestLineId from server
    2. Add action types:
       - `CACHE_FRAME`: `{ type: "CACHE_FRAME"; anchorLineId: number; lines: TermLine[] }` -- store frame in cache
       - `SET_SCROLL_STATE`: `{ type: "SET_SCROLL_STATE"; anchorLineId: number | null; newestLineId: number | null }` -- update anchor/newest tracking
       - `CLEAR_ANCHOR`: `{ type: "CLEAR_ANCHOR" }` -- client requests return to live mode
    3. Reducer:
       - `CACHE_FRAME`: set frameCache.set(anchorLineId, lines), return new state with cloned map.
       - `SET_SCROLL_STATE`: update anchorLineId and newestLineId.
       - `CLEAR_ANCHOR`: set anchorLineId = null. Do NOT clear frameCache (cache persists for re-scroll).
       - `SET_TERMINAL_LINES`: if anchorLineId is null (live mode), update lines as before. If anchored, do NOT update lines (live frames should not overwrite scrolled view -- though frame-pusher should not send them, this is defense in depth).
    4. Initialize frameCache as `new Map()`, anchorLineId as null, newestLineId as null.

    **chat/index.tsx changes:**
    1. In the `terminal_frame` handler for full frames:
       - If `frame.anchorLineId` is present (scrolled frame from server):
         a. Dispatch `CACHE_FRAME` with anchorLineId and frame.lines.
         b. Dispatch `SET_SCROLL_STATE` with frame.anchorLineId and frame.newestLineId.
         c. Dispatch `SET_TERMINAL_LINES` with frame.lines (to show on screen).
       - If `frame.anchorLineId` is absent (live frame):
         a. Only dispatch `SET_TERMINAL_LINES` if `terminalStateRef.current.anchorLineId === null` (live mode). Otherwise skip -- user is scrolled, don't overwrite.
         b. Dispatch `SET_SCROLL_STATE` with anchorLineId=null and newestLineId from frame (if present).
    2. In `handleTerminalScroll`:
       - Before sending scroll request to relay, check client cache:
         a. Compute expected anchorLineId based on direction and delta from current state.
         b. If `terminalState.frameCache.has(expectedAnchorLineId)`, dispatch SET_TERMINAL_LINES from cache immediately (no network needed). Also dispatch SET_SCROLL_STATE.
         c. Still send the scroll request to server (server needs to track anchor state).
       - For "down" direction: if scrolling would go past newestLineId (anchor + rows > newest), dispatch CLEAR_ANCHOR and request a terminal_frame_request for fresh live frame.
    3. Add prefetch logic:
       - After receiving a scrolled terminal_frame (anchorLineId present), send 1-2 additional scroll requests for adjacent anchors that aren't cached. Use requestAnimationFrame or setTimeout(0) to avoid blocking.
       - Prefetch up: anchorLineId - rows (one page up from current).
       - Prefetch down: anchorLineId + rows (one page down, unless it would be live).
       - Only prefetch if not already cached.

    **terminal-viewport/index.tsx changes:**
    1. Add `isScrolled: boolean` prop (derived from `anchorLineId !== null` in chat/index.tsx).
    2. When `isScrolled` is true, render a small "Scrolled -- tap to return" indicator at the bottom of the viewport (simple View + Text, positioned absolute at bottom center, translucent dark background, white text).
    3. Add `onTapToReturn` prop callback. When indicator is tapped, call it. In chat/index.tsx, the handler dispatches CLEAR_ANCHOR and sends terminal_frame_request for live frame.
    4. Wire the new props from chat/index.tsx.
