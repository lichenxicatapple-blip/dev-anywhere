# Quick Task 260415-k44: Migrate feishu client from Taro mini program to pure React SPA + PWA - Context

**Gathered:** 2026-04-15
**Status:** Ready for planning

<domain>
## Task Boundary

将 apps/feishu（Taro 飞书小程序）迁移到 apps/web（纯 React SPA + PWA），去掉 Taro 抽象层，使用标准 Web API，部署为飞书网页应用。

</domain>

<decisions>
## Implementation Decisions

### 新项目位置
- monorepo 内新建 `apps/web`，与 proxy、relay、shared 并列
- `apps/feishu` 保留但不再维护，作为存档参考
- 复用 `@cc-anywhere/shared` 包，pnpm workspace 直接引用

### CSS 和响应式方案
- 使用 Tailwind CSS，替代 Taro 的 750px 设计稿 pxtransform
- 响应式断点：mobile(<768px) / tablet(768-1024px) / desktop(>1024px)
- 暗色主题使用 Tailwind dark: 前缀
- 现有 CSS 文件中的样式迁移到 Tailwind utility classes

### UI 组件层
- 使用 shadcn/ui 作为基础组件库（源码复制模式，底层 Radix UI + Tailwind）
- 替换现有自定义 Toast、Modal、Button 等通用组件
- 初始化时配置项目主题色（深色背景 #1E1E1E、强调色 #00D4AA）
- 迁移阶段先替换通用组件，业务组件保持现有视觉
- 迁移跑通后再统一间距、字号、圆角等设计 token

### PTY 终端渲染
- 用 xterm.js 替换自写 TerminalViewport 栅格渲染组件
- proxy 保留 @xterm/headless + serialize addon 用于快照（commit b05bec2 有历史实现）
- proxy 转发原始 PTY 字节流，同时喂给 headless xterm 维护状态
- relay 简化：透传二进制帧，移除 terminal_frame / scroll_request / anchor 逻辑
- 客户端 scrollback 由 xterm.js 本地管理，移除服务端滚动架构
- 可删除大量代码：TerminalTracker、frame cache、anchor scroll、delta 压缩
- **需要详细设计的异常场景：**
  - 断线重连：快照 + 增量重放（proxy 定期用 serialize addon 生成快照，重连时发送最近快照 + 快照后的原始字节）
  - 快照和事件流必须持久化到磁盘（恢复 b05bec2 的 EventStore 设计），支持 proxy 重启后恢复、session 历史回看
  - 多客户端同时观看同一 session 的字节流广播
  - 网络慢时的背压控制和字节流积压
  - UTF-8 / ANSI 转义序列在帧边界截断的处理
  - 高速输出时的流控（手机端 xterm.js 渲染性能）
  - WebSocket 传输一步到位用 binary frame，不搞 base64+JSON 过渡

### 迁移策略
- 26 个零 Taro 依赖的文件不盲目 copy，逐个 review 后再迁入，顺带清理技术债

### 路由和部署
- relay 服务器同进程 serve 静态文件，一个进程同时提供 WebSocket + 前端
- hash 路由（单 index.html），无需服务端路由配置
- 开发阶段用 Vite dev server + WebSocket proxy 到 relay

### 未来功能考量
- PWA 支持 Screen Wake Lock API（屏幕常亮）
- PWA 支持 Web Speech API（语音朗读 + 语音识别）
- 这些能力在小程序中不可用，进一步验证迁移方向正确

</decisions>

<specifics>
## Specific Ideas

- Taro 组件机械替换：View→div, Text→span, ScrollView→div overflow, Image→img
- 业务逻辑层（stores, services, message-parser, relay-store）零改动复用
- 浏览器原生 overflow-anchor 解决 load-more 滚动锚定问题
- 浏览器原生 element.scrollIntoView() 解决自动滚底问题
- Vite + React + TypeScript + Tailwind CSS 作为技术栈
- react-router 处理路由（hash mode）
- marked + highlight.js 继续用于 Markdown 渲染，但可以用 dangerouslySetInnerHTML（浏览器环境）

</specifics>
